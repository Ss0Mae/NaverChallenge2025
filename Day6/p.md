# **학습 목표**

- 작은 역할을 하는 클래스부터 조금씩 더 복잡한 클래스를 만들면서 객체지향 설계와 객체지향 프로그래밍 패러다임을 연습하는 것이 목표다.
- 각 역할을 담당하는 클래스를 별도 파일로 구현할 수 있다.
- 더 작은 단위나 반복되는 코드가 있다면 일반화해서 별도 클래스로 구현해야 한다.

## **왜 필요한가?**

객체지향 프로그래밍은 이제 대부분의 프로그래밍 언어에 구현되어 있고, 많은 개발자들이 사용하는 보편적인 프로그래밍 패러다임이다.

클래스 단위로 일반적인 내용을 추상화하고, 더 구체적인 클래스를 만드는 방법을 학습하는 것은 중요하다.

객체지향 프로그래밍 방식으로 코드를 구조화하는 것은 쉽지 않지만, 재사용성과 생산성을 향상시키고 유지보수 효율성을 높이는 방법이다.

# **기능요구사항**

### **게임 배경**

마블 인기 캐릭터 종류와 타노스가 각자 자신의 보드에 배치하고 컴퓨터와 1 : 1로 진행하는 보드게임을 객체지향 프로그래밍 방식으로 설계하고 구현해야 한다. 상대편 보드는 서로 보이지 않는다고 가정한다.

![](https://lucas-image.codesquad.kr/1751539139453_2025-board.png.png)

### **핵심 기능**

- Thanos, BlackWidow(BW), Hulk(HK), DrStrainge(DS), SpiderMan(SP), Loki(LK), Thor(TR) 클래스를 각각 다른 파일로 저장한다.
- Board 클래스를 반드시 따로 구현해야 한다.
- 파일을 추가하거나 기능을 추가할 때마다 소스 파일을 gist에 커밋한다.
- 프로그램을 구현할 때 입력 → 검증 → 처리/계산 → 형식 → 출력 단계를 구분한다.
    - 각 단계를 담당하는 객체 혹은 모듈로 분리한다.
- 프로그램을 시작하면 랜덤하게 8개 캐릭터를 고르고 위치에 배치한다.
    - 타노스는 항상 2개를 A-C 사이 아무 위치에나 배치해야 한다.
    - BW, DS, TR는 최소한 1개씩 배치해야 한다.
    - TR와 SM는 최대 1개만 배치가 가능하다.
    - BW, HK는 최대 2개까지 DS와 LK는 최대 3개까지 배치가 가능하다.
    - 타노스를 제외한 캐릭터는 D-E 사이에 배치해야 한다.
- 프로그램을 동작하는 동안 사용자와 컴퓨터가 반복해서 입력한다.
- 공격하는 나의 캐릭터와 상대편 목표 지점을 차례로 입력받아서 공격한다.
    - 입력값은 기존 위치(FROM) 문자와 **`>`** 화살표, 그리고 공격하려는 위치(TO) 2자리 문자를 입력받는다.
        
        예시) **`A1->B2`** : A1에 캐릭터로 B2를 이동하거나 공격한다.
        
- 입력 형식에 맞지 않거나 자신의 차례가 아닌 경우는 다시 입력받는다.
- 만약 공격한 곳에 상대편 캐릭터가 있다면 HIT를 출력하고, 없다면 MISS를 출력하고 이동한다. 다만 캐릭터마다 움직일 수 있는 규칙은 다르며 규칙에 따라야 한다.
    - HIT 공격 성공일 경우 공격포인트만큼 HP가 감소한다.
    - 만약 HP가 0이 되면 해당 캐릭터를 사라진다.
- 입력값이 **`?`**물음표를 입력하면 상대편 전체 HP를 확인할 수 있다.

### **보드게임 요구사항**

- Thanos 체력(HP)은 **`1000`**으로 시작하고, 공격하면 상대편은 **`100`**씩 감소한다.
- 다른 마블 캐릭터 체력과 공격 포인트는 다음과 같다.
    - BlackWidow : HP **`400`**, 공격 **`10`**
    - Hulk : HP **`800`**, 공격 **`50`**
    - DrStrainge : HP **`700`** , 공격 **`40`**
    - Loki : HP **`500`**, 공격 **`30`**
    - SpiderMan : HP **`600`**, 공격 **`20`**
    - Thor : HP **`900`**, 공격 **`50`**
- 공격한 곳에 캐릭터가 있으면 해당 캐릭터 HP가 감소한다.
- 만약 HP가 0보다 작아지면 해당 캐릭터는 보드에서 제거한다.

# **프로그래밍 요구사항**

### **게임 구성**

게임은 서로 다른 보드를 두 개 가지고 서로 공격하도록 구성해야 한다.

입력은 사용자가 먼저 입력하고 컴퓨터는 그 다음에 입력한다.

### **보드 구성**

- Board는 각각 5x6 (가로:행 row x 세로:열 column) 크기 캐릭터 존재 여부를 관리한다.
- Board는 모든 말의 위치를 간접적으로 알 수 있어서 전체를 출력할 수 있다. 출력을 위한 display() 함수는 적절한 데이터 구조로 A행부터 E행까지 전체를 행 단위로 리턴한다.
    - 출력을 위한 문자열 배열로 리턴하면 안 되고, 필요한(문자열이 아닌 타입을 포함하는) 데이터 구조를 리턴한다.
    - Board에서 return한 데이터 구조를 바탕으로 출력 형식을 담당하는 객체(혹은 모듈)에서 문자열 배열로 바꾸고 출력한다.
    - Thanos은 두 칸을 차지하고 모두 표시하고, 다른 마블 캐릭터는 약자 두 글자를 표시한다.
    - 아래 예상 출력 참고
 ### **캐릭터 초기 생성 테스트하기**

- 특정 위치에 특정 캐릭터를 생성하는 기능을 테스트한다.
    - 이름과 매개변수는 스스로 판단해서 작성한다.
    - 초기 위치가 아니면 생성하지 않는지 확인한다.
    - 이미 해당 위치에 다른 말이 있으면 생성하지 않아야 한다.
    - 캐릭터 종류별로 최대 개수보다 많이 생성할 수는 없다.
    - 생성하지 않는 경우는 exception 예외처리로 상위에서 어떤 예외상황인지 판단한다.

### **특정 캐릭터 무조건 생성하기**

- 특정 위치에 특정 캐릭터를 생성하는 기능을 테스트한다.
    - 이름과 매개변수는 스스로 판단해서 작성한다.
    - 바로 직접 조건 생성함수와 다르게 어떤 캐릭터든지 특정 위치에 놓을 수 있어야 한다.
    - 이미 해당 위치에 다른 말이 있으면 생성하지 않는다.
    - 최대 개수도 고려하지 않는다.

### **특정 위치 이동 또는 공격하기**

- 입력받은 특정 위치를 입력해서 공격할 수 있어야 한다.
    - 내가 공격하려는 from 위치에 캐릭터를 찾고
    - 공격하는 to 위치에 캐릭터가 있을 때는 공격한다.
    - 해당 자리에 캐릭터가 있으면 규칙에 따라 남은 HP를 반환하고, HP가 0이 되거나 위치에 캐릭터가 없으면 0을 반환한다.
    - 만약 해당 자리에 없으면 이동한다.

### **캐릭터 공통**

- 모든 캐릭터는 위치값을 저장할 Position 타입을 활용한다.
    - 꼭 Position 값을 다루기 위한 데이터 구조를 별도로 만든다.
- 캐릭터는 편이 확실하게 구분되어야 하며 한 번 생성하면 바뀌지 않는다.
- 타노스를 제외한 캐틱터들은 각자 색깔 스톤을 가질 수 있다.
    - 모든 스톤은 각각 1개씩이며 WHITE, BLUE, RED, GREEN, ORANGE, VIOLET 6가지 종류가 있다.
    - 여러 인스턴스가 있더라도 단 하나의 인스턴스 가진다.
    - 만약 공격을 당하고 나서 HP가 초기값보다 절반 이하가 되면 색깔 스톤을 빼앗긴다.

### **Thanos 요구사항**

- 타노스는 어느 위치든지 이동하거나 공격할 수 있다.
- 타노스가 특정 위치로 가려면 다른 한 칸까지 두 칸이 비어 있어야 이동할 수 있다.
- HP는 **`1000`**, 공격력은 **`100`**
- 만약 특정 캐릭터를 공격해서 해당 캐릭터 HP가 초기 대비 절반 이하이고, 해당 캐릭터가 스톤을 가지고 있으면 빼앗긴다.
- 스톤은 타노스 인스턴스 2개가 서로 공유한다.
- 스톤을 뺏을 때마다 HP는 +50 되고 공격력은 +10 된다.

### **Black Widow 요구사항**

- 이동할 때는 전, 후, 좌, 우 1칸씩만 이동할 수 있다.
- 만약 다른 말이 있으면 이동할 수 없다.
- HP는 **`400`**, 공격력은 **`10`**
- 특정 인스턴스는 **`Orange`** 스톤을 가질 수 있다.

### **게임 종료 조건**

각자 나만의 방식으로 게임 종료 조건을 명시하고 구현한다.

예를 들어 **`모든 캐릭터 체력이 0이 되면 끝난다`** 처럼 종료 조건을 만들고 해당 조건이 완성될 경우 게임을 종료한다.

- 최종 실행 결과에 게임 종료 화면을 캡처해서 첨부한다.
