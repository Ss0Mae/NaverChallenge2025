### 💻 1단계: 프로젝트 기본 구조 설정
- [X] `Position` 클래스 구현: 좌표(row, col)를 관리하고, 문자열 기반 좌표(A1)와 상호 변환 기능을 포함합니다.
- [X] `Character` 추상 클래스 정의: 모든 캐릭터가 공통으로 가질 속성(HP, 공격력, 팀)과 메서드(move, attack)의 기반을 마련합니다.
- [X] `Board` 클래스 기본 구조 구현: 5x6 그리드를 생성하고, 특정 위치의 캐릭터 정보를 가져오는 기능을 포함합니다.
- [X] 각 캐릭터(`Thanos`, `BlackWidow` 등) 클래스 파일을 생성하고 `Character`를 상속받도록 기본 구조를 설정합니다.

### 🎲 2단계: 게임 초기화 기능 구현
- [X] `Board` 클래스에 캐릭터를 배치하는 `placeCharacter` 메서드를 구현합니다.
- [X] 캐릭터 종류별 생성 규칙(필수 포함, 최대 개수)을 검증하는 로직을 추가합니다.
- [X] 게임 시작 시, 요구사항에 맞춰 8개의 캐릭터를 랜덤하게 생성하고 보드에 배치하는 초기화 로직을 구현합니다.

### ⌨️ 3단계: 입출력 및 게임 루프 구현
- [X] 사용자 입력을 받고, `A1->B2` 형식인지 유효성을 검사하는 입력 처리 모듈을 구현합니다.
- [X] `Board`가 전달한 데이터 구조를 바탕으로, 현재 보드 상태를 콘솔에 출력하는 출력 처리 모듈을 구현합니다.
- [X] 플레이어와 컴퓨터가 번갈아 턴을 진행하는 기본 게임 루프를 구현합니다.

### 💥 4단계: 핵심 게임 로직 (이동 및 공격)
- [X] 각 캐릭터의 고유한 이동 규칙에 따라 `move` 메서드를 구체적으로 구현합니다.
- [X] `HIT`와 `MISS`를 판정하고, `HIT`일 경우 상대방의 HP를 감소시키는 공격 로직을 구현합니다.
- [X] HP가 0 이하로 떨어진 캐릭터를 보드에서 제거하는 로직을 구현합니다.

### ✨ 5단계: 고급 기능 구현
- [X] `Stone` 클래스 또는 관련 로직을 구현하고, 캐릭터가 스톤을 소유하거나 빼앗기는 기능을 추가합니다.
- [X] `Thanos`가 스톤을 획득했을 때 능력치가 강화되는 특수 능력을 구현합니다.
- [X] `?`를 입력했을 때 상대방의 총 HP를 보여주는 기능을 구현합니다.
- [X] 직접 정의한 게임 종료 조건을 확인하는 로직을 구현합니다.
