# 📝 나만의 체크포인트 (Day 13-14)

## 🎯 목표: Git 내부 구조 분석 도구 개발

### 1️⃣ 단계: 프로젝트 설계 및 환경 설정

- [x] **짝과 함께 목표 정의**: 우리 팀이 만들 분석 도구의 최종 목표와 결과물 형태 합의하기
- [x] **핵심 기능 우선순위 결정**: `add`, `commit`, `log` 중 어떤 순서로 분석 기능을 구현할지 정하기
- [ ] **기본 아키텍처 설계**: 모듈(e.g., `GitRepository`, `ObjectParser`, `IndexParser`)의 역할과 책임 분담하기
- [x] **개발 환경 구축**: `package.json`을 생성하고, `zlib`, `crypto` 등 필요한 Node.js 모듈 학습 계획 세우기

### 2️⃣ 단계: Git 객체(Object) 분석 기능 구현

- [x] **객체 파일 경로 찾기**: SHA 해시값(`ab12...`)을 `.git/objects/ab/12...` 형태의 파일 경로로 변환하는 로직 구현하기
- [x] **객체 압축 해제**: `zlib` 모듈을 사용하여 객체 파일의 압축을 풀고 내용을 읽는 함수 구현하기
- [x] **객체 파싱**: 압축 해제된 객체 내용에서 헤더(타입, 크기)와 실제 데이터를 분리하는 로직 구현하기
- [ ] **`cat-file` 흉내내기**: 특정 해시값에 해당하는 객체의 내용을 예쁘게 출력하는 기능 구현하기 (첫 목표는 `blob` 객체)

### 3️⃣ 단계: `add` & `commit` 명령어 분석 기능 구현

- [ ] **`add` 분석**:
    - [x] **Blob 객체 생성 확인**: `git add` 후 새로 생긴 blob 객체를 찾아 내용 확인하기
    - [ ] **Index 파일 분석 (도전 과제)**: `.git/index` 바이너리 파일의 구조를 학습하고, `add` 이후 변경된 파일 정보를 읽어내는 파서(Parser) 구현하기
- [ ] **`commit` 분석**:
    - [x] **Tree 객체 분석**: 커밋이 가리키는 `tree` 객체를 재귀적으로 탐색하여 특정 시점의 파일 목록 출력하기
    - [x] **Commit 객체 분석**: 커밋 객체를 파싱하여 부모 커밋, 저자, 커밋 메시지 등 상세 정보 출력하기

### 4️⃣ 단계: `log` 명령어 재현 및 리팩토링

- [x] **참조(Refs) 분석**: `.git/HEAD` 파일을 읽어 현재 브랜치를 알아내고, `.git/refs/heads/[브랜치명]` 파일을 읽어 최신 커밋 해시 가져오기
- [x] **커밋 히스토리 순회**: 최신 커밋에서 시작하여, 각 `commit` 객체의 `parent` 정보를 이용해 부모 커밋을 연쇄적으로 찾아가는 로직 구현하기
- [x] **`log` 출력**: 순회하며 얻은 커밋 정보를 실제 `git log`와 유사한 형태로 터미널에 출력하기
- [ ] **코드 리팩토링**: 중복되는 로직을 함수로 추출하고, 각 모듈의 역할을 명확하게 다듬어 코드 품질 향상시키기

### 5️⃣ 단계: 코드 구조 개선 (리팩토링)

- [ ] **모듈 설계**: `my-git.js`의 기능을 `GitRepository`, `ObjectManager`, `IndexManager`, `RefsManager` 등 역할 기반의 모듈로 분리하는 아키텍처 설계하기
- [ ] **`ObjectManager` 구현**: 객체 읽기(`readObject`), 쓰기(`hashObject`) 관련 로직을 별도 모듈로 분리하기
- [ ] **`IndexManager` 구현**: `readIndex`, `writeIndex`, `updateIndex` 등 인덱스 관련 로직을 별도 모듈로 분리하기
- [ ] **`RefsManager` 구현**: `getHeadCommit`, `getHeadRef`, `updateHead` 등 참조 관련 로직을 별도 모듈로 분리하기
- [ ] **CLI 진입점(`my-git-cli.js`) 재구성**: 각 모듈을 가져와 사용자의 명령어에 따라 실행 흐름을 제어하도록 `switch` 문 리팩토링하기

### 6️⃣ 단계: `clone` 명령어 원리 구현 (로컬 복제)

- [ ] **`clone` 명령어 추가**: `my-git-cli.js`에 `clone <source-path> <destination-path>` 명령어를 처리하는 로직 추가하기
- [ ] **객체 전체 복사**: 원본 저장소의 `.git/objects` 디렉토리 내 모든 파일을 대상 저장소로 재귀적으로 복사하는 기능 구현하기
- [ ] **참조(브랜치) 복사**: 원본 저장소의 `.git/refs/heads` 내 브랜치 파일들을 대상 저장소에 복사하는 기능 구현하기
- [ ] **`HEAD` 설정**: 복제 후, 대상 저장소의 `HEAD`가 원본의 기본 브랜치를 가리키도록 설정하는 기능 구현하기
- [ ] **워킹 디렉토리 체크아웃 (도전 과제)**: `HEAD` 커밋의 `tree` 객체를 분석하여, 실제 파일들을 워킹 디렉토리에 생성해주는 `checkout` 로직 구현하기
