# **학습 목표**

- Git 내부 동작을 이해하고, Git object 저장 구조를 만들면서 배울 수 있다.
- 파일 시스템을 이해하고, 소스 관리를 위해 필요한 필수 지식을 학습할 수 있다.
- 짝 설계까지만 함께 고민하고, Git 동작을 함께 학습하면서 목표를 정한다.

# **사전지식**

없음

# **기능요구사항**

- 버전 관리를 위해 **`.git`** 디렉토리 아래에 있는 파일들을 분석하는 도구를 만들어야 한다.
- Node 기반 콘솔 명령 프로그램을 작성한다.
    - sha256 이나 zlib는 node 패키지를 활용한다.
- 분석해야 하는 Git 명령어 종류는 다음과 같다. 짝끼리 명령어별로 우선순위를 정해보자.
    - **`init`**: 필요한 작업 디렉토리를 생성하고 초기화할 때
    - **`branch`**: 브랜치 목록을 보거나 생성하거나 지울 때
    - **`add`**: 디렉토리의 변경된 파일들을 스테이지(index)에 추가할 때
    - **`status`**: 작업 트리의 상태를 출력할 때
    - **`commit`**: 스테이지 파일들을 커밋으로 기록할 때
    - **`log`**: 커밋 로그를 출력할 때
    - **`switch`**: 스테이지 파일들을 커밋으로 기록할 때

### **짝 설계 요구사항**

- 한 명이 독점해서 설계하지 않는다
- 두 명 이상이 함께 하나의 의도를 가지고 작은 것부터 설계에 참여한다
- 전체적인 목표와 큰 작업 단위를 우선 협의하는 것을 권장한다
- 모든 세부적인 목표를 모두 협의하거나 결정하려고 하지 않아도 된다

# **프로그래밍 요구사항**

### **주요 git object**

- 오브젝트와 인덱스에 대해 학습하고, 읽어서 처리 가능한 프로그램을 작성한다.
    - 오브젝트 디렉토리: **`디렉토리명/.git/objects/`** 또는 윈도우 **`디렉토리명\.git\objects\`**
    - 인덱스 파일: **`디렉토리명/.git/index`** 또는 윈도우 **`디렉토리명\.git\index`**
    - 로그 파일: **`디렉토리명/.git/logs/`** 또는 윈도우 **`디렉토리명\.git\logs/`**

다음 명령들은 반드시 동작을 확인할 수 있도록 구현해보자.

모든 경우를 대비해서 구현하기보다는, 필요한 데이터나 명령의 동작을 확인하기 위해 일감을 작은 단위로 나눠서 해결해보자.

### **add 명령어**

- **`git add`** 명령으로 파일을 추가할 때 어떤 데이터가 바뀌는지 학습하고, 직접 프로그램으로 확인할 수 있도록 구현한다.

### **commit 명령어**

- **`git commit`** 명령으로 커밋을 추가하면 오브젝트가 어떻게 바뀌는지 확인하고, 직접 프로그램으로 확인할 수 있도록 구현한다.

### **log 명령어**

- **`git log`** 명령으로 보여주는 데이터를 확인하고, 비슷하게 표시하려면 어떻게 해야 하는지 직접 프로그램으로 확인할 수 있도록 구현한다.

# **예상결과 및 동작예시**

파일을 바꿔가면서 Git 명령으로 내부 데이터가 어떻게 바뀌는지, 스스로 만든 도구로 확인하며 결과를 출력해보자. 출력 형식은 짝이 함께 결정한다.
