# **벡터DB 만들기 개선하기 🆙**

# **학습 목표**

- 벡터데이터베이스에 필요한 데이터를 분석하고 기준 함수를 적용해서 벡터 데이터를 생성할 수 있다
- 벡터데이터베이스 동작을 점검하기 위해서 테스트 코드를 적용하고 구조를 개선할 수 있다
- 벡터데이터베이스 쿼리 속도를 개선할 수 있다
- 벡터데이터베이스 기본 동작에 대해 성능을 분석하고 공간복잡도 관점에서 메모리 사용을 최적화할 수 있다
- 벡터데이터베이스 기본 동작에 대해 성능을 분석하고 시간복잡도 관점에서 CPU 사용을 최적화할 수 있다

# **사전지식**

# **기능요구사항**

### **데이터 생성 요구사항**

https://www.kaggle.com/competitions/quora-question-pairs/data 에 있는 **`test.csv`** 파일 데이터를 다운로드받아 순차적으로 읽어서 question1, question2 에 있는 단어들끼리 관련도 - 노출빈도 - 거리를 벡터형태로 변환해서 저장해본다

- 이렇게 데이터를 벡터로 계산해서 저장할 때 필요한 것은 무엇일까

### **성능 진단**

CPU 사용률, 메모리 사용률을 진단하고 분석할 수 있는 프로파일링 도구를 찾아보고 기본 동작 중에 CPU와 메모리 사용률을 분석해본다.

- 분석한 병목 지점 혹은 사용이 과한 지점을 찾아서 개선하고 해당 부분이 이전/이후로 어떻게 달라지는 지 테스트 코드로 점검한다

# **프로그래밍 요구사항**

### **벡터 데이터 생성**

- 제공한 데이터 파일을 벡터 DB 데이터로 저장해서 해당 질문끼리 유사도를 검증할 수 있도록 활용한다고 가정한다
- kaggle 에 있는 문제 그대로 중복인지 아닌지 **`is_duplicate`**를 예측하거나 판단하는 모델을 만들어야 하는 것은 아니다. 의미있는 혹은 유용한 벡터값을 제대로 저장하는 것에 집중해보자

### **성능 진단과 최적화**

직접 만든 벡터DB의 성능을 진단해보고 개선할 부분을 찾아서 개선해야 한다

- CPU 사용률을 측정하고 출력해보자
- 메모리 사용률을 측정하고 출력해보자
- 테스트 코드로 사용률을 점검해보자
- 개선할 지점을 찾아내고 개선하고 다시 기존 테스트 코드로 비교해보자
