# **기능요구사항**

## **학습**

- 벡터 데이터베이스가 익숙하지 않다면 먼저 유명한 벡터 데이터베이스를 사용해 보고 사용방법을 먼저 학습한다.
- 많이 사용하는 유명한 벡터 DB에 대해 조사를 해 보고 동작방식 (저장방식, 검색 알고리즘, 인덱싱 여부 등)에 대해 정리한다
- **`word2vec`** 동작 방식에 대해 학습한다

## **벡터 데이터베이스 구현**

다음 기능을 만족하는 벡터 데이터베이스를 구현해야 한다.

- 데이터베이스를 생성한다. 이 때 저장할 벡터의 차원을 지정한다.
- 생성한 데이터베이스에 벡터 데이터를 추가하고 삭제할 수 있다.
- 벡터 데이터는 (id, vector) 의 형태를 가진다.
- 데이터를 추가할 경우 해당 데이터는 영속성을 가진다.
- 벡터에 대한 knn 쿼리를 수행할 수 있다.

## **성능 평가**

- 데이터가 정확하게 저장되는지 검증한다.
- 쿼리 속도에 대한 평가 방법을 고민해 보고 성능 평가를 수행한다.
- 쿼리 정확성에 대한 평가 방법을 고민해 보고 정확성 평가를 수행한다.

## **간단한 응용 프로그램 작성**

- 작성한 데이터베이스를 가지고 직접 간단한 실제 예제 프로그램을 작성해서 정상적으로 동작하는지 확인한다.

### **짝 설계 요구사항**

- 한 명이 독점해서 설계하지 않는다
- 두 명 이상이 함께 하나의 의도를 가지고 작은 것부터 설계에 참여한다
- 전체적인 목표와 큰 작업 단위를 우선 협의하는 것을 권장한다
- 모든 세부적인 목표를 모두 협의하거나 결정하려고 하지 않아도 된다. 결정한 사항과 결정하지 못한 사항을 구분해서 기록해도 좋다
- 짝끼리 설계한 도식이나 설명을 readme 문서에 반드시 기록한다

# **프로그래밍 요구사항**

## **설계**

- 구현을 하기 전에 충분히 고민을 하고 적합한 구조와 동작방식을 고려하여 설계를 해야한다.

## **벡터 데이터베이스 구현**

- 기능 요구사항을 만족해야 한다.
- 외부 라이브러리나 도구 사용을 최소화한다.

## **성능 평가**

- 차원과 데이터 크기를 어떻게 해야 할 것인지 고민해 본다.
