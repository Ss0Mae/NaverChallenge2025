# **학습 목표**

- 동영상 서비스에서 동시에 여러 고객이 여러 영상을 업로드했을 때, 등록부터 변환하고 검증할 때까지 여러 모듈이 진행하는 비동기 작업을 시뮬레이션하는 프로그램을 만드는 것이 전체 미션 목표다.
- 이번 단계에서는 하나의 서버에 업로드부터 후처리하는 흐름을 구현한다.

## **왜 필요한가?**

스레드를 무한정 생산하는 방식으로 멀티 스레드 구현은 어렵다. 그래서 여러 프로그램들이 단일 스레드에서도 이벤트 단위의 병렬 처리 구조로 동작한다. 스레드를 무한정 늘리지 않고 비동기 방식으로 동작하는 병렬 처리 방식에 대해 이해하고 구현할 수 있어야 한다.

# **기능요구사항**

- 분석과 설계 단계에서 설계 결과를 기록한다. 우선 학습 저장소에 설계 결과를 추가한 상태에서 개발하기 시작해야 한다.
- node 기반으로 콘솔에서 동작하는 프로그램을 작성한다.
- 지금까지 미션들에서 학습했던 다양한 패턴, 응용 방식을 활용해서 비동기로 동작하도록 작업한다.
- 다음 요구사항에 나오는 구성 요소들을 객체나 모듈로 구분해서 구현한다.
- 동기 함수들 보다 비동기 함수를 중심으로 구현한다.
- 구현 단계를 진행할 때 마다 gist에 commit을 기록한다.

# **프로그래밍 요구사항**

- 이번 단계 구현은서버가 하나면서, 모듈도 1개라고 가정하고 작업한다.
- 검증 작업이 끝나면 바로 **`공개중`** 상태로 바꾼다.
- 1단계 작업 전체가 아니라 구성 요소를 구현될 때마다 gist에 커밋한다.

### **1단계. 기본 흐름**

- 직접 스레드를 만들면 안되고, Event를 처리하는 비동기 이벤트 처리하도록 구현한다.
- 등록 모듈은 동영상 업로드 등록을 연속해서 받을 수 있다.
    - 등록 요청을 받으면 등록 대기큐(Queue)에 추가한다.
- 대기큐에 들어있는 이벤트를 꺼내서 처리하는 이벤트 루퍼(Looper)를 별도 모듈/객체로 분리해서 구현한다.
- 업로드 매니저(Manager)는 주기적으로 등록 대기큐에서 이벤트를 확인한다.
- 등록 대기를 확인하고 업로드가 완료되면, 영상 변환 모듈 큐에 이벤트를 전달한다.
    - 이 때 필요하면 매니저(Manager)도 진행 현황판을 표시할 이벤트를 전달할 수 있다.
    - 업로드하는 시간 자체는 무시하고 완료되는 시점부터 시작한다.
- 변환 모듈은 한 번에 1개 영상을 변환할 수 있다고 가정한다.
    - 스레드를 생성하는 게 아니라 이벤트 큐를 받아서 처리하는 방식으로 동작해야 한다.
    - 변환 작업을 시작할 때와 끝날 때 마다 이벤트를 발생한다.
    - 영상 길이마다 변환 시간이 다르다.
        - 단편 : 3분, 중편 : 7분, 장편 : 15분
- 라이세스 검증 모듈은 한 번에 1개까지 영상만 검증할 수 있다고 가정한다.
    - 라이센스 검증은 영상 길이와 상관없이 동일하게 10분씩 걸린다고 가정한다.
    - 스레드를 생성하는 게 아니라 이벤트 전달 방식으로 동작해야 한다.
    - 검증 모듈도 배송 시작할 때와 끝날 때 마다 이벤트를 발생한다.
- 진행 현황판(DashBoard)는 전체 변환 현황을 주기적으로 화면에 출력한다.
    - 등록한 영상에 대해 **`대기중`**/**`변환중`**/**`검증중`**/**`공개중`** 상태를 표시한다.
    - 현황판도 이벤트를 받아서 처리하는 별도 모듈/객체로 분리해서 구현한다.
- 아래 동작 예시는 이해를 돕기위한 예시일 뿐 동일하게 출력할 필요 없다. 각 요구사항을 확인하기 위해서 매니저, 대기큐, 현황판 등을 이벤트 큐와 비동기 방식으로 어떤 구조로 구분하고, 어느 모듈에서 출력할 지 고민한다.
- 동작 확인을 위해서 분단위 분류 시간을 실제로는 초단위로 처리해도 무방하다.

# **예상결과 및 동작예시**

서버가 1개이고 변환 모듈 1개, 검증 모듈이 1개만 있다고 가정한 상태 예시

```
> 영상 업로드  =  1. 단편(3분)    2. 중편(7분)    3. 장편(15분)
> 업로드할 영상을 입력하세요. 예) 단편 2개 => 1:2
> 1:2
대기중/1,1/
단편 변환 시작
:1 대기중/1/
.(1분경과)

> 3:1
:1 대기중/1,3/
..(2분경과)
단편 변환 후 검증 시작
단편 변환 시작
..(2분경과)

> 2:2
:1 대기중/3,2,2/
.(1분경과)
단편 검증 대기
:3 대기중/2,2/
장편 변환 시작
.....(7분경과)
단편 검증 완료 -> 공개중
단편 검증 시작
......(8분경과)
:2 대기중/2/
중편 변환 시작
..(2분경과)
단편 검증 완료 -> 공개중
....(5분경과)
장편 검증 시작
중편 변환 시작
:2 대기중//
....(7분경과)
//
장편 검증 완료 -> 공개중
...(3분경과)
중편 검증 시작
... (중간생략)
모든 영상 공개했습니다.
```
