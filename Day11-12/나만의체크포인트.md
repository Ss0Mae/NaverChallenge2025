# 📝 나만의 체크포인트

## 1️⃣ 단계: 기본 파이프라인 설계 및 구현 (Mission 1)

- [X] **개념 설계**: 각 모듈(등록, 루퍼, 매니저, 변환, 검증, 대시보드)의 역할과 책임을 명확히 정의하기
- [X] **이벤트 정의**: 모듈 간 통신에 사용할 이벤트 이름과 전달할 데이터 구조 설계하기 (e.g., `new_upload`, `conversion_start`, `validation_complete`)
- [X] **Queue 구현**: 등록 대기, 변환 대기, 검증 대기를 처리할 큐(Queue) 자료구조 구현하기
- [X] **핵심 모듈 구현**: `EventEmitter`를 활용하여 각 모듈의 기본 로직 구현하기
- [X] **시간 시뮬레이션**: `setTimeout`과 `Promise`를 사용하여 영상 변환 및 검증 시간 구현하기
- [X] **대시보드 구현**: 주기적으로(`setInterval`) 현재 모든 영상의 상태를 출력하는 기능 구현하기
- [X] **통합 및 테스트**: 모든 모듈을 연결하고, 단일 영상이 파이프라인을 정상적으로 통과하는지 테스트하기

## 2️⃣ 단계: 파이프라인 확장 및 고도화 (Mission 2)

- [X] **동적 모듈 생성**: 프로그램 시작 시 입력받은 개수만큼 변환/검증 모듈 인스턴스 생성하기
- [X] **로드 밸런싱 로직 구현**: 매니저가 각 모듈의 작업 부하를 추적하고, 가장 한가한 모듈에 작업을 할당하는 로직 구현하기
- [X] **동시 처리 능력 향상**: 변환 모듈이 동시에 2개의 작업을 처리할 수 있도록 로직 수정하기
- [X] **전담 처리 기능 구현**: 특정 변환 모듈이 특정 영상 종류(단편/중편/장편)를 우선 처리하도록 로직 구현하기
- [ ] **고객별 그룹핑**: 고객 ID를 기준으로 업로드된 영상들을 그룹으로 묶어 관리하는 자료구조 설계하기
- [ ] **고객별 현황판 개선**: 대시보드에 고객별로 진행 상황을 시각적으로 구분하여 표시하기
- [ ] **최종 완료 처리**: 특정 고객의 모든 영상 처리가 완료되었을 때 특별한 알림을 주는 기능 구현하기
- [ ] **자동 종료 로직**: 모든 작업이 완료되었는지 확인하고, 안전하게 프로그램을 종료하는 로직 구현하기

## CS 배경지식 학습

- [X] **이벤트 루프 (Event Loop)**: Node.js가 어떻게 싱글 스레드로 동시성을 처리하는지 내부 동작 원리 학습하기
- [X] **프로세스와 스레드**: 스레드를 직접 생성하는 방식과 이벤트 기반 방식의 차이점과 장단점 비교 분석하기
- [X] **디자인 패턴**: Pub/Sub 패턴, 옵저버 패턴, 싱글톤 패턴 등 이번 미션에 적용할 수 있는 디자인 패턴 학습하기
- [X] **자료구조**: 큐(Queue)의 동작 방식(FIFO)과 JavaScript에서의 효율적인 구현 방법 학습하기
