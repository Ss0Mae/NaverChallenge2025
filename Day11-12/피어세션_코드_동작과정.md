## 1. 데이터 생성 흐름 (`generate-data.js`)

이 스크립트의 목표는 단순히 랜덤 데이터를 만드는 것을 넘어, **검색의 의미와 현실성을 부여하는 구조화된 데이터셋**을 구축하는 것입니다.

> **❓ 질문: 모든 벡터를 랜덤으로 생성하는데 어떻게 "쌍문동" 맛집이 정확하게 검색되나요?**
> 
> **💡 답변: 개별 벡터는 랜덤이지만, 벡터 간의 '관계'는 의도적으로 설계되었기 때문입니다.**
> 모든 맛집 벡터는 완전히 무작위로 생성되는 것이 아닙니다. 특정 **'대표 벡터'** 를 기준으로, 그와 **'가까운 위치'** 에 있도록 생성됩니다. 이 과정을 통해 벡터 공간에 자연스러운 **군집(Cluster)** 이 형성되고, 이것이 의미 있는 검색을 가능하게 하는 핵심 원리입니다.

아래에서 그 과정을 자세히 살펴보겠습니다.

### **1단계: 설정 및 헬퍼 함수 정의**

스크립트 상단에서 데이터 생성에 필요한 기본값들을 설정합니다.

```javascript
const VECTOR_DIMENSION = 8;       // 벡터의 차원 수
const CATEGORIES = { ... };         // '쌍문동', '강남역' 등 카테고리 정의
const ITEMS_PER_CATEGORY = 10;    // 카테고리별 순수 데이터 수
const FUSION_ITEMS_PER_MIX = 3; // 퓨전 데이터 수
const NOISE_ITEMS = 10;           // 완전 무작위 데이터 수
```

또한, 벡터 연산을 위한 3개의 핵심 헬퍼 함수를 정의합니다.
- `createRandomVector()`: 정규화된(크기가 1인) 랜덤 벡터를 생성합니다.
- `addNoise()`: 원본 벡터에 약간의 노이즈를 추가하여 비슷하지만 다른 벡터를 만듭니다.
- `blendVectors()`: 두 벡터를 특정 비율로 섞어 '퓨전' 특성을 가진 벡터를 만듭니다.

### **2단계: 카테고리별 '대표 벡터' 생성 (기준점 설정)**

가장 먼저, 사용자가 입력할 검색어(`쌍문동`, `강남역` 등)를 대표하는 **기준 벡터(Anchor Vector)** 를 생성합니다. 이 벡터들은 n차원 벡터 공간에 찍히는 **'기준점'** 이 됩니다. 이 대표 벡터 자체는 `createRandomVector()`로 생성되므로 완전히 무작위입니다. 하지만 일단 한번 생성되면, 이 값은 변하지 않는 상수가 됩니다.

```javascript
// keywords.json 에 저장될 객체
const keywords = {};

for (const categoryName in CATEGORIES) {
    // 각 카테고리 이름(예: "쌍문동")에 대해 랜덤 벡터를 생성하여 할당
    keywords[categoryName] = createRandomVector(VECTOR_DIMENSION);
}
```
- **핵심:** `keywords['쌍문동']` 벡터와 `keywords['강남역']` 벡터는 서로 다른 랜덤 값을 가지므로, 공간적으로 멀리 떨어져 있습니다.

### **3단계: '관계'를 주입하여 맛집 데이터 생성 (3가지 타입)**

이제 앞서 만든 **대표 벡터와의 관계**를 기반으로, 실제 맛집 벡터들을 생성합니다. 이것이 검색을 가능하게 하는 가장 중요한 과정입니다.

**A. 순수 데이터 (Pure Data) - "가깝게 만들기"**
- 각 카테고리의 **대표 벡터**에 `addNoise()` 함수로 **아주 약간의 노이즈만 추가**하여 맛집 벡터를 생성합니다.
- **결과**: '쌍문동 맛집 1', '쌍문동 맛집 2' 등의 벡터들은 모두 `keywords['쌍문동']`이라는 기준점 바로 근처에 옹기종기 모여있게 됩니다. 자연스럽게 **'쌍문동 클러스터'** 가 형성됩니다.
- **목적**: '쌍문동'으로 검색(즉, `keywords['쌍문동']` 벡터로 검색)했을 때, 이 데이터들이 가장 높은 유사도로 검색되도록 하여 검색의 정확성을 보장합니다.

**B. 퓨전 데이터 (Fusion Data) - "중간 지점에 만들기"**
- `blendVectors()` 함수를 사용해 **'쌍문동' 대표 벡터와 '강남역' 대표 벡터를 7:3 비율로 섞습니다.**
- **결과**: 이렇게 생성된 '퓨전 맛집' 벡터는 '쌍문동' 기준점과 '강남역' 기준점 사이의 어딘가에, 하지만 '쌍문동' 쪽에 더 가깝게 위치하게 됩니다.
- **목적**: '쌍문동'으로 검색했을 때, 이 퓨전 맛집이 순수 맛집보다는 낮은 순위로, 하지만 완전한 '강남역' 맛집보다는 높은 순위로 검색되게 하여 현실성을 더합니다.

**C. 노이즈 데이터 (Noise Data) - "아무데나 만들기"**
- 어떤 대표 벡터와도 관계없이, `createRandomVector()`로 완전히 새로운 랜덤 벡터를 생성합니다.
- **결과**: 이 벡터들은 벡터 공간의 아무 곳에나 흩어져 존재하게 됩니다.
- **목적**: 검색 결과에 예상치 못한 데이터가 매우 낮은 유사도로 포함되게 하여, 실제 검색 환경과 유사한 노이즈를 재현합니다.

### **4단계: 파일 시스템에 저장**

이렇게 **'관계'가 부여된** 모든 벡터 데이터를 `fs.writeFileSync`를 통해 3개의 JSON 파일로 저장합니다. 이로써 `main.js`에서 사용할 수 있는, 의미 있는 검색이 가능한 데이터셋 준비가 완료됩니다.
- `keywords.json`: 검색의 기준점이 되는 대표 벡터
- `vectors.json`: 대표 벡터와의 관계를 기반으로 생성된 모든 맛집 벡터
- `metadata.json`: 맛집의 이름, 카테고리 등 부가 정보

## 2. CLI 실행 흐름 (`main.js`)

이 스크립트는 사용자와 직접 상호작용하며, `VectorDB` 모듈을 활용해 실제 검색을 수행합니다.

### **1단계: 초기화 및 데이터 로드**

애플리케이션이 시작되면, 가장 먼저 필요한 모듈과 데이터를 메모리로 불러옵니다.

```javascript
const VectorDatabase = require('./VectorDB');
const fs = require('fs');
const readline = require('readline');

// VectorDB 인스턴스 생성
const db = new VectorDatabase('./vectors.json', './metadata.json');
// 키워드 데이터 로드
const keywords = loadKeywords('./keywords.json');
```
- `new VectorDatabase(...)`가 호출되는 순간, `VectorDB` 클래스의 `constructor`가 실행되어 `vectors.json`과 `metadata.json`의 모든 데이터를 읽어 `db.vectors`와 `db.metadata`에 저장해 둡니다.

### **2단계: 대화형 인터페이스 시작**

Node.js의 `readline` 모듈을 사용하여 사용자 입력을 받을 수 있는 CLI 환경을 구성합니다.

```javascript
const rl = readline.createInterface({ ... });

function startCLI() {
    rl.question('VDB> ... ', (userQuery) => {
        // ... 입력 처리 로직 ...
    });
}

startCLI(); // 최초 실행
```
- `rl.question` 함수가 프롬프트를 띄우고 사용자 입력을 비동기적으로 기다립니다.

### **3단계: 사용자 입력 처리 및 검색 벡터 준비**

사용자가 검색어를 입력하고 엔터를 누르면, 콜백 함수가 실행됩니다.

```javascript
// userQuery는 사용자가 입력한 문자열 (예: "강남역")
if (userQuery.toLowerCase() === 'exit') { ... }

// 1단계에서 로드한 keywords 객체를 사용
const queryVector = keywords[userQuery];

if (!queryVector) {
    // 사용 가능한 검색어가 아니면 에러 메시지 출력
    console.log(...);
    startCLI(); // 다시 입력 대기
    return;
}
```
- 이 단계의 핵심은, **사용자의 텍스트 입력을 미리 약속된 벡터(`queryVector`)로 변환**하는 것입니다. 실제 Word2Vec 모델을 사용하는 대신, `keywords.json`을 조회하는 방식으로 이를 시뮬레이션합니다.

### **4단계: 핵심 검색 실행 (`VectorDB.search`)**

준비된 `queryVector`를 가지고, `VectorDB`의 `search` 메서드를 호출하여 핵심 검색 로직을 수행합니다.

```javascript
const searchResults = db.search(queryVector, 5);
```
- `search` 메서드 내부에서는 다음과 같은 일이 순차적으로 일어납니다.
    1.  `db.vectors`에 저장된 모든 벡터 데이터를 `map`으로 순회합니다.
    2.  각 벡터에 대해 `this.cosineSimilarity(queryVector, item.vector)`를 호출하여 유사도 점수를 계산합니다.
    3.  `{ id: ..., score: ... }` 형태의 객체 배열이 만들어집니다.
    4.  이 배열을 `score` 기준으로 내림차순 정렬합니다.
    5.  `slice(0, k)`를 통해 상위 k개의 결과만 잘라내어 반환합니다.

### **5단계: 결과 포맷팅 및 출력**

`search`로부터 받은 ID와 점수 리스트를 사용자가 보기 좋게 가공합니다.

```javascript
searchResults.forEach((result, index) => {
    // db.metadata에서 ID로 실제 맛집 이름과 정보를 조회
    const metadata = db.metadata[result.id];
    if (metadata) {
        console.log(`${index + 1}) ${metadata.name} ...`);
    }
});
```

### **6단계: 다음 입력 대기 (루프)**

모든 결과 출력이 끝나면, 함수 마지막에서 `startCLI()`를 다시 호출하여 다음 사용자 입력을 기다리는 상태로 돌아갑니다. 이 재귀적인 호출 구조를 통해 사용자가 `exit`를 입력하기 전까지 프로그램이 계속 실행됩니다.
