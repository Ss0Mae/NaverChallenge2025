# **학습 목표**

- 쉘 스크립트 명령을 활용해서 특정 파일을 읽고, 값을 비교해서, 다른 명령을 실행하도록 스크립트로 자동화할 수 있다.
- ollama 방식으로 sLLM 모델을 직접 실행해서 활용할 수 있다.

### **왜 필요한가?**

- 리눅스 명령을 조금이라도 사용해보거나 쉘 스크립트로 데이터를 자동화해서 수집하는 업무는 모든 분야 개발자가 해야하는 일 중에 하나이다. 이제 작은 모델 정도는 직접 연동할 수 있어야 한다.

# **사전지식**

- 리눅스 기본 명령에 대한 사전 지식이 필요합니다.

# **기능요구사항**

- 다음 각 단계를 진행하면서 작업한 명령어, 스크린 캡처, 진행 과정, 실행 결과를 README.md에 추가한다.

### **1단계. ollama 설치**

- 가상 머신에 설치한 리눅스에 **`ollama`** 를 설치한다.
  - 공식 사이트 주소 https://github.com/ollama/ollama
  - 리눅스를 재부팅해도 자동으로 실행되도록 ollama 관련 환경값을 설정한다.
  - 반드시 가상머신 외부에서 접근 가능하도록 설정한다.
  - ollama 에 **`gemma3:1b`** 모델을 실행한다. (성능과 여유 공간이 더 있다면 다른 모델을 활용해도 되지만 가장 작은 모델로도 충분하다)
  - 모델 실행 이후 외부에서 curl 같은 네트워크 명령으로 동작을 확인한다.

### **2단계. 주기적인 데이터 수집**

- 가상 머신에 설치한 리눅스에서 주기적으로 데이터를 수집하는 쉘 스크립트를 작성한다. 수집한 데이터는 **`/monitoring`** 디렉토리에 스스로 정한 파일 형식으로 저장한다.
  - 어떤 데이터를 얼마 주기로 수집할 것인가 스스로 결정한다.(아래 예시보다 다양한 선택지를 찾아보고 선택가능하며, 대신 적어도 분 단위로 계속해서 업데이트되는 데이터를 추천한다.)
  - 예시) 시스템 로그: syslog 또는 dmesg

### **3단계. 데이터 비교와 분석**

- 2단계에서 주기적으로 수집한 데이터를 1시간마다 비교하고 분석한다.
  - 예시) dmesg 로그를 1시간마다 분석해서 가장 빈번한 이벤트가 무엇이고, 그 이벤트가 자주 발생하는 이유를 ollama 모델에 전달해서 200자 이내로 요약한다.
- 이렇게 ollama로 실행하는 모델로 해당 데이터 구조나 발생 빈도, 의미있는 내용을 분석한 결과를 챌린지 채널로 전송하는 쉘 스크립트로 작성한다.

# **프로그래밍 요구사항**

### **가상 환경 요구사항**

- 가상 머신에 설정하는 것도 본인 컴퓨터 CLI 환경에서 접속해야 한다.
- 주기적인 데이터 수집은 crontab 을 활용한다.
- 챌린지 서버로 전송할 때는 다음과 같은 방식으로 전송한다.
  - 반드시 본인 캠퍼ID를 포함한다.
  - 30분 내에 30회 이상 보내지 않는다.

```
curl -X POST -H 'Content-type: application/json' --data '{"text":"J041 CPU 사용률이 높습니다. 그 이유는 빈번하게 커널 작업을 실행하기 때문입니다"}' https://hooks.slack.com/services/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

```

- 이 웹훅 주소는 저장소나 공개된 곳에 링크를 올리면 슬랙에서 막혀 버립니다. 테스트를 위해서 전송해도 되지만 동일한 내용을 자주 보내면 해킹으로 의심하고 정지되기 때문에 주의해 주세요.
- 알림을 보낼 때 마다 **`/monitoring`** 폴더에 **`YYYYMMDD-HHMMSS`** 파일 이름 형식으로 생성하고 알림 메시지 내용을 기록한다.
- ollama 모델 동작 중에 메모리 사용률은 개발자가 지속적으로 감시한다.

> 앞으로 자신이 학습한 내용을 소스 코드 뿐만 아니라, 설계나 결과를 확인할 수 있는 자료도 함께 저장하는 습관을 만든다.
