## 🧭 1. 스스로 확인할 사항 (개인 학습 나침반)

> 챌린지 동안 문제 해결 과정 중 학습한 내용에 대해 스스로 점검하고, 배경 지식과 경험을 정리해 보세요.

### **1) 리눅스와 유닉스의 역사와 차이점**

- **학습 목표:** 우리가 사용하는 Ubuntu(리눅스)가 어떤 뿌리에서 시작되었는지 이해하고, 유닉스와의 관계를 통해 OS의 큰 그림을 파악합니다.
- **탐구 질문:**
    - 유닉스는 어떻게 탄생했으며, 어떤 철학(Philosophy)을 가지고 있었나요? (예: "모든 것은 파일이다")
    - 리눅스는 유닉스와 어떤 관계인가요? "유닉스 계열(Unix-like)"이라는 말은 무슨 뜻일까요?
    - macOS도 유닉스와 관련이 깊다고 하는데, 어떤 점에서 그럴까요?

### **2) 다양한 쉘(Shell) 종류와 명령어 옵션**

- **학습 목표:** 우리가 매일 사용하는 쉘의 종류와 특징을 이해하고, `bash`와 `zsh`를 중심으로 더 효율적인 명령어 사용법을 익힙니다.
- **탐구 질문:**
    - `sh`, `bash`, `zsh`는 각각 어떤 특징과 차이점이 있나요? 왜 많은 개발자들이 `bash` 대신 `zsh`와 `oh-my-zsh`를 사용할까요?
    - `ls -al`, `rm -rf`처럼 명령어 뒤에 붙는 `-` (하이픈)은 무엇을 의미하며, 어떻게 이런 옵션들을 효과적으로 찾아보고 학습할 수 있을까요? (`man` 명령어 활용법)
    - 이번 미션에서 사용한 `find`, `grep`, `awk`, `sed` 같은 텍스트 처리 명령어들은 어떻게 다른가요? 언제 어떤 명령어를 쓰는 것이 가장 효율적일까요?

### **3) 리눅스 디렉토리 권한과 소유권**

- **학습 목표:** `drwxr-xr-x`와 같은 권한 표기법을 완벽히 이해하고, `chmod`, `chown`을 이용해 파일과 디렉토리의 접근을 제어하는 원리를 체득합니다.
- **탐구 질문:**
    - `rwx` 권한이 파일(file)에 적용될 때와 디렉토리(directory)에 적용될 때, 그 의미는 어떻게 다른가요? (예: 디렉토리의 `x` 권한)
    - `chmod 755`와 `chmod u=rwx,go=rx`는 같은 의미입니다. 숫자 표기법과 문자 표기법의 장단점은 무엇일까요?
    - 이번 미션에서 `test` 계정이 `dmesg`를 실행하기 위해 `sudoers` 파일을 수정했습니다. 이것은 일반적인 권한 관리와 어떻게 다른, 더 특별한 케이스일까요?

## 🗺️ 2. 다같이 확인할 사항 (팀 토론 지도)

> 챌린지 문제 해결 과정 이후, 동료와 서로 다른 시각에서 설명하고 이해한 내용을 공유해 보세요. 배경 지식과 본인의 의견을 함께 정리하여 토론에 참여합니다.

### **1) Ollama 생태계 탐험과 활용 방안**

- **탐구 목표:** Ollama를 단순히 "로컬 AI 실행 도구"를 넘어, 어떤 프로젝트들과 함께 사용되며 어떤 새로운 가능성을 만들어내고 있는지 탐색합니다.
- **토론 주제:**
    - **Ollama + LangChain/LlamaIndex:** Ollama를 LangChain 같은 LLM 애플리케이션 프레임워크와 연동하면 무엇을 할 수 있을까요? (예: RAG - 검색 증강 생성)
    - **Ollama Web UI:** `Open WebUI` 같은 프로젝트는 Ollama에 어떤 편의성을 더해주나요? 이런 UI 도구가 있다면 우리의 미션 수행 과정이 어떻게 달라졌을까요?
    - **Beyond Chat:** 채팅 외에, Ollama를 우리 주변의 문제를 해결하는 데 어떻게 활용해볼 수 있을까요? (코드 생성, 데이터 분석, 문서 요약 등)

### **2) Ollama 실행 환경 비교: 가상 머신(VM) vs. 로컬(Local)**

- **탐구 목표:** 이번 미션에서 VM에 Ollama를 설치한 경험을 바탕으로, 로컬 PC에 직접 설치하는 경우와의 장단점을 비교하고 트레이드오프를 이해합니다.
- **토론 주제:**
    - **자원 격리 vs. 성능:** VM을 사용하면 시스템 자원이 격리되어 안정적이지만, 성능 손실이 발생할 수 있습니다. 로컬에 직접 설치하면 성능은 좋지만, 내 PC의 다른 작업에 영향을 줄 수 있습니다. 어떤 상황에 어떤 방식이 더 유리할까요?
    - **네트워크 설정:** VM에서 외부 접속을 허용하기 위해 방화벽과 네트워크 설정을 변경했습니다. 로컬 PC에서 실행했다면 이 과정이 어떻게 달라졌을까요? `localhost`와 `0.0.0.0`의 차이점은 무엇일까요?
    - **개발 환경의 일관성:** 팀 프로젝트를 진행한다고 가정해 봅시다. 모든 팀원이 VM을 사용하는 것과 각자 로컬 PC에 설치하는 것 중, 어떤 방식이 개발 환경의 일관성을 유지하는 데 더 도움이 될까요?
