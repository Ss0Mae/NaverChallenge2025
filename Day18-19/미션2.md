# **학습 목표**

- 고객과 MD 클라이언트 끼리 코드 재사용 가능성을 높일 수 있다
- 클라이언트와 서버를 통합해서 테스트할 수 있다
- 클래스/파일/패키지 (또는 모듈) 단위로 구분해서 진단하고 개선할 수 있다
- 테스트 가능한 구조를 넘어서 유지보수 하기 쉬운 구조를 고려해서 개선할 수 있다

# **사전지식**

없음

# **기능요구사항**

- 고객과 MD 클라이언트 코드를 재사용성 기준을 정하고 수치로 측정해보자
- 클라이언트와 서버 코드를 재사용성 기준을 정하고 수치로 측정해보자
- 클라이언트와 서버를 직접 실행하지 않고 통합 테스트 코드로 양쪽을 통합해서 테스트한다
    - 테스트하기 적합한 동작 또는 시나리오 범위를 스스로 정한다
- 테스트 가능하면서도 유지보수가 쉬운 구조를 고려해서 개선한다
- 10KB 이상 긴 메시지를 보내고 받을 수 있도록 개선한다

# **프로그래밍 요구사항**

- 3주차 day11부터 경험했던 개선하기 경험들을 종합해서 판단해보자
- 특히 타입이나 클래스 단위테스트도 하면서 클라이언트와 서버를 합쳐서 진행하는 통합테스트까지 고려해서 개선해야 한다
- 고객 클라이언트와 MD 클라이언트의 공통 로직이 있다면 일반화하거나 모듈화해서 분리하고 단위 테스트를 작성한다
- (기능 추가보다는) 구조적인 측면에서 제약사항을 벗어나서 한계나 제한 사항을 없애도록 개선한다
