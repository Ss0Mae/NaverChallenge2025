## 1. TCP/IP 와 UDP의 핵심 차이

이번 미션은 TCP와 UDP를 모두 사용하므로, 두 프로토콜의 근본적인 차이를 이해하는 것이 가장 중요합니다.

| 구분 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
| --- | --- | --- |
| **연결 방식** | **연결 지향 (Connection-oriented)** | **비연결 지향 (Connectionless)** |
|  | 통신 전, 3-way-handshake로 연결을 설정. | 연결 설정 없이 데이터그램을 바로 전송. |
| **신뢰성** | **높음** | **낮음** |
|  | 데이터 순서 보장, 유실 시 재전송. | 순서 보장 안됨, 재전송 없음. |
| **속도** | 상대적으로 느림 | 빠름 |
| **미션 활용** | `login`, `buy` 등 **신뢰성이 중요한** 모든 요청/응답 | `broadcast` 등 **신속함이 중요하고 일부 유실되어도 괜찮은** 메시지 전송 |

## 1.1 TCP 추가 이해: 내부 메커니즘 심화

- **흐름 제어 (Flow Control)**
    - 슬라이딩 윈도우(Window) 및 수신 윈도우 크기 조정 원리
    - Node.js에서 `socket.setNoDelay()`와 `socket.setKeepAlive()` 옵션 활용
- **혼잡 제어 (Congestion Control)**
    - Slow Start, Congestion Avoidance 단계 개념
    - 패킷 손실 시 스루풋 변화 예상
- **타임아웃과 재전송**
    - 손실된 세그먼트에 대해 TCP가 재전송을 트리거하는 방식
    - 과제에서 “로그인 후 응답이 없을 때 타임아웃” 로직 설계

## 2. Node.js `net`과 `dgram` 모듈

Node.js는 TCP와 UDP 통신을 위한 내장 모듈을 제공합니다.

- **`net` 모듈 (TCP):**
    - `net.createServer()`: TCP 서버를 생성합니다. 클라이언트가 접속(`connection`)할 때마다 소켓 객체를 전달받아 1:1 통신을 시작합니다.
    - `net.createConnection()`: TCP 클라이언트를 생성하여 서버에 접속합니다.
    - **소켓(Socket):** 서버와 클라이언트 간의 통신 창구. `data`, `close`, `error` 등의 이벤트를 통해 데이터를 주고받고 상태를 관리합니다.
- **`dgram` 모듈 (UDP):**
    - `dgram.createSocket('udp4')`: UDP 소켓을 생성합니다.
    - `socket.bind(PORT)`: 특정 포트에서 UDP 메시지를 수신 대기합니다.
    - `socket.send(msg, port, address)`: 지정된 주소와 포트로 메시지를 전송합니다.

## 3. 소켓 통신을 위한 메시지 프로토콜 설계

HTTP와 달리 TCP 소켓 통신은 정해진 데이터 형식이 없습니다. 따라서 직접 통신 규칙(프로토콜)을 설계해야 합니다.

### 왜 필요한가?

- TCP는 데이터를 스트림(Stream)으로 처리하므로, 어디까지가 하나의 완전한 메시지인지 구분할 방법이 필요합니다. (TCP의 경계 문제)
- 예를 들어, 서버가 "HELLO"와 "WORLD"를 연달아 보내면 클라이언트는 "HELLOWORLD" 또는 "HEL", "LOWORLD" 처럼 데이터를 조각내어 받을 수 있습니다.

### 해결 방안 (미션 프로토콜)

1. **헤더(Header) 추가:** 메시지 본문의 길이를 알려주는 `Content-Length` 헤더를 추가합니다.
2. **본문(Body) 형식 정의:** 실제 데이터는 `JSON` 형식으로 통일하여, 다양한 구조의 데이터를 쉽게 파싱하고 확장할 수 있도록 합니다.
3. **구분자(Delimiter) 사용:** 헤더와 본문, 그리고 각 메시지의 끝을 명확히 구분하기 위해 `\r\n\r\n`과 같은 특정 문자열을 사용합니다.

## 3.1 메시지 프레이밍 심화

- **버퍼 언패킹(Stateful Parser)**
    - TCP 스트림 파싱 시, 헤더/본문이 분리돼 도착할 때의 처리 흐름
    - Node.js `Buffer` 객체와 `accumulator` 패턴 활용 예시 코드
- **헤더 확장**
    - `Message-Type: login` 같은 커스텀 헤더 추가로 라우팅 로직 간소화
    - JSON Body 외에 바이너리 페이로드(이미지 전송 등) 고려

## 4. 동시성 및 상태 관리

- **동시성 (Concurrency):** 서버는 여러 클라이언트의 요청을 동시에 처리해야 합니다. Node.js는 이벤트 기반 비동기 I/O 모델을 사용하므로, 각 클라이언트의 소켓 이벤트(`data`, `close` 등)를 독립적으로 처리하는 방식으로 동시성을 관리합니다.
- **상태 관리 (State Management):** 서버는 각 클라이언트의 상태를 메모리에 저장하고 관리해야 합니다.
    - *누가 로그인했는가? (`campId`)
    - *어떤 그룹에 속해 있는가? (`groupId`)
    - *현재 채팅 중인가?
    - *상품 재고는 몇 개인가?
- **클러스터(Cluster) 모드**
    - `cluster` 모듈을 활용해 멀티코어 환경에서 서버 확장
- **세션 스토리지 추상화**
    - Redis나 Map 기반 분리 설계로 확장성 확보
- **비정상 연결 종료 처리**
    - `socket.timeout` 이벤트 활용한 세션 클린업 시나리오

## 7. 유니캐스트, 브로드캐스트, 멀티캐스트

- **유니캐스트 (Unicast):** 1:1 통신. TCP 통신은 기본적으로 유니캐스트입니다. 미션의 `snapchat` 기능은 특정 그룹원에게만 메시지를 보내므로, 서버가 그룹 정보를 바탕으로 수동으로 구현하는 유니캐스트의 한 형태입니다.
- **브로드캐스트 (Broadcast):** 네트워크에 연결된 모든 장치에 메시지를 전송. 미션의 `broadcast` 기능은 UDP를 사용하여 구현합니다.
- **멀티캐스트 (Multicast):** 특정 그룹에 속한 장치들에게만 메시지를 전송. 이번 미션에서는 직접 다루지 않지만, 알아두면 좋은 개념입니다.
