# ✅ Day4: 나만의 체크포인트

## 🎯 최종 목표: 프로세스 메모리 모델 시뮬레이터 구현

### **Phase 1: 🏗️ 시뮬레이터의 뼈대 세우기**

- [x] **1-1. `Simulator` 클래스 기본 구조 설계:**
  - `constructor`에서 TEXT, STACK, HEAP 영역을 나타낼 `Buffer` 또는 `ArrayBuffer`를 정해진 크기만큼 할당합니다.
  - `PC(Program Counter)`, `SP(Stack Pointer)` 등 핵심 상태 변수를 초기화합니다.
  - 타입별 크기를 저장할 `typeSizes` 맵(Map) 객체를 생성합니다.
- [x] **1-2. `reset()` 함수 구현:**
  - 모든 메모리 영역을 0으로 채우고, 상태 변수(PC, SP 등)를 초기값으로 되돌리는 기능을 구현합니다. `constructor`에서 `reset()`을 호출하여 초기화를 위임하는 것도 좋은 방법입니다.
- [x] **1-3. `setSize(type, length)` 함수 구현:**
  - `typeSizes` 맵에 타입 이름과 크기를 저장합니다.
  - 이미 등록된 타입은 수정할 수 없도록 예외 처리를 합니다.

---

### **Phase 2: 🧠 핵심 메모리 관리 기능 구현 (내부 함수)**

- [x] **2-1. 스택 `push`/`pop` 내부 함수 구현:**
  - `push(value)`: `SP`를 증가시키고 스택 영역에 값을 쓰는 함수를 만듭니다. (요구사항의 `OOOOOOOO(address)`)
  - `pop()`: 스택에서 값을 읽고 `SP`를 감소시키는 함수를 만듭니다. (요구사항의 `OOOOOOOO()`)
- [x] **2-2. `alloc(type, count)` 내부 함수 구현:**
  - `typeSizes`를 참조하여 필요한 총 메모리 크기를 계산합니다. (패딩 규칙 포함)
  - 힙 영역에서 비어있는 공간을 찾아 할당하고, 시작 주소를 반환합니다. (간단한 `heapPointer`를 이동시키는 방식부터 시작)
  - 할당된 힙 주소를 스택에 `push`합니다.
- [x] **2-3. `free(stackAddress)` 내부 함수 구현:**
  - `stackAddress`가 유효한 스택 주소인지 확인합니다.
  - 스택에서 힙 주소를 읽어온 뒤, 해당 힙 주소가 유효한지 확인합니다.
  - 힙 영역의 해당 공간을 '사용 가능'으로 표시합니다. (메모리 관리 방식에 따라 구현이 달라짐)
  - 유효하지 않은 주소일 경우 예외를 발생시킵니다.

---

### **Phase 3: 📜 어셈블리 명령어 해석 및 실행**

- [x] **3-1. `locate(funcName, codes)` 함수 구현:**
  - 함수 이름과 코드 시작 위치(TEXT 영역 인덱스)를 맵에 저장합니다.
  - 코드를 TEXT 영역에 순서대로 저장합니다.
- [x] **3-2. `다음()` (실행 엔진) 및 `VAR` 명령어 구현:**
  - `PC`가 가리키는 TEXT 영역의 코드를 한 줄 읽어옵니다.
  - `VAR $VarName : $Type` 또는 `VAR $VarName : $Type[$Count]` 구문을 파싱(parsing)합니다.
  - 파싱 결과에 따라 `alloc()`을 호출하여 메모리를 할당합니다.
- [x] **3-3. `CALL` / `RETURN` 명령어 구현:**
  - **`CALL`**:
    - 복귀 주소(`PC + 1`)를 스택에 `push`합니다.
    - 호출 스택(callstack) 관리를 위한 정보를 기록합니다.
    - `PC`를 호출할 함수의 시작 주소로 점프시킵니다.
  - **`RETURN`**:
    - 현재 함수의 지역 변수들이 사용하던 스택 공간을 정리합니다.
    - 스택에서 복귀 주소를 `pop`하여 `PC`에 설정합니다.
    - `$RETURN` 값이 있으면 스택에 임시 저장했다가 다음 명령어에서 사용하도록 처리합니다.
- [x] **3-4. `SET` / `RELEASE` 명령어 구현:**
  - **`SET`**: 변수 이름(과 인덱스)에 해당하는 힙 주소를 찾아 값을 씁니다.
  - **`RELEASE`**: 변수 이름으로 스택 주소를 찾아 `free()`를 호출합니다.

### **Phase 4: 📊 상태 확인 유틸리티 구현**

- [x] **4-1. `usage()` 함수 구현:**
  - 스택과 힙의 전체 크기, 현재 사용량, 남은 용량을 계산하여 반환합니다.
- [x] **4-2. `callstack()` 함수 구현:**
  - `CALL` 시 기록해 둔 호출 스택 정보를 바탕으로 `함수명() @주소 -> ...` 형태의 문자열을 만들어 반환합니다.
- [ ] **4-3. `heapdump()` 함수 구현:**
  - 현재 할당된 힙 영역을 순회하며, 각 블록의 정보(타입, 주소, 크기, 참조하는 스택 주소 등)를 보기 좋은 문자열로 만들어 반환합니다.

---

### **Phase 5: ✨ 고급 기능 및 마무리**

- [ ] **5-1. `garbageCollect()` 함수 구현:**
  - **Mark**: 스택 전체를 스캔하여, 참조되고 있는 모든 힙 주소를 "살아있음"으로 표시합니다.
  - **Sweep**: 힙 전체를 스캔하여, "살아있음" 표시가 없는 메모리 블록을 `free` 처리합니다.
- [ ] **5-2. 테스트 케이스 실행 및 디버깅:**
  - 요구사항에 명시된 예시 코드를 실행하며 각 단계의 동작을 확인하고, `heapdump`, `callstack` 등의 유틸리티로 내부 상태를 검증하며 버그를 잡습니다.
