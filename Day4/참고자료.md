### **섹션별 역할**

### **TEXT 섹션**

- 프로그램에 있는 함수 코드, 제어문, 상수 등을 포함한다.
- 일반적으로 한 번 로딩하면 바뀌지 않는다.
    - JVM에서는 TEXT 영역을 사용하지 않는다. JVM 메모리 구조 설명 자료
        
        https://dev.to/deepu105/visualizing-memory-management-in-jvm-java-kotlin-scala-groovy-clojure-19le
        
    - node나 브라우저에서는 TEXT 영역 대신 코드 영역이 별도로 존재한다.

### **GVAR/BSS 섹션**

- 범위(Scope)가 정해지지 않는 전역(Global 또는 Static) 변수를 포함한다.
- GVAR 는 초기값을 0이 아닌 특정한 값으로 지정한 경우 사용한다.

### **HEAP 섹션**

- 동적으로 할당되는 메모리 공간으로 malloc 이나 new 명령으로 할당한다.
- 힙 영역에 할당한 메모리 공간에 대한 주소를 참조하는 경우가 많다.

### **STACK 섹션**

- 함수를 호출할 때마다 지역 변수, 매개변수와 리턴값 등이 쌓인다.
