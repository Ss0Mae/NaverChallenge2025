# Day4

# **학습 목표**

- 프로세스 메모리 관리 모델에 대해 알아보고 가상 메모리와 프로세스별 메모리 사용 현황을 분석하는 방법을 학습한다.
- 스택Stack 영역, 힙Heap 영역, 텍스트Text 영역, 글로벌변수GVAR 영역의 역할과 동작 방식을 이해하고 기본적인 동작 방식을 구현하는 것을 목표로 한다.

### **왜 필요한가?**

- 현대 운영체제들은 프로세스를 실행할 때마다 프로세스 메모리 모델을 생성해서 관리한다. 메모리 관리 모델에 대한 지식을 학습하고 동작 방식을 이해하면 모든 프로그래밍 언어 동작과 처리 흐름를 이해하는 데 도움이 된다.

# **사전지식**

- 운영체제별 가상 메모리 관리와 프로세스 메모리 구조에 대한 학습이 필요하다.

# **프로그래밍 요구사항**

분석한 프로세스 메모리 구조를 바탕으로 비슷하게 동작하는 시뮬레이션하는 프로그램을 작성한다. 다음 내용은 설계를 도와주는 내용을 포함하고 있다.

- 다음과 같은 영역을 최소한 크기를 정해서 미리 확보한다.
    - STACK 영역 : 어셈블리 코드를 실행하면서 호출할 때마다 필요한 값을 스택 방식으로 Push하거나 Pop하는 영역
    - HEAP 영역 : 어셈블리 코드를 실행하면서 메모리 할당 요청이 있을 때 사용할 공간을 확보하는 영역
    - TEXT 영역 (문자열배열) : 입력으로 주어지는 어셈블리 코드가 저장되는 영역
        - 예를 들어 명령어 한 줄(배열 인덱스 1개)이 4바이트씩이라고 가정한다.
- 스택에서 힙 영역에 메모리 주소를 다룰 때는 **`포인터`** 변수 개념을 직접 구현해야 한다.
- 프로그램에서 처리하는 모든 포인터 메모리 사이즈는 4바이트를 기준으로 한다.
    - 힌트 : 언어에서 4바이트를 기준으로 동작하는 타입을 활용한다.
- 일반적으로 동작하도록 설계하고 구현한다.
    - 시나리오 흐름에 맞는 예시 프로그램을 작성하고, 실행해 본다.
    - 다양한 경우에 대한 동작을 확인하기 위한 시나리오 흐름은 스스로 결정한다.

### **시뮬레이터 클래스**

다음과 같이 동작하는 함수를 포함하는 **`Simulator`** 객체를 구현한다.

- TEXT, STACK, HEAP 영역은 크기에 맞게 미리 할당한다.
- TEXT 영역에서 몇 번째 명령을 실행중인지 **`PC(Program Counter)`** 변수를 사용한다.
- STACK 영역을 위해서 **`Stack Pointer`**변수를 두고 관리한다.
- HEAP 영역을 관리하는 방식은 스스로 판단한다.
    - 이번 미션에서는 일반적인 프로세스 메모리 모델(배경 지식 참조)중에서 텍스트, 스택과 힙 영역을 위주로 구현한다. 다른 영역은 무시하고 구현해도 된다.

### **𝔸. 함수 요구사항**

- 매개변수나 리턴에 필요한 타입은 스스로 판단해서 선언한다.
- 리턴 명세가 있는 경우는 반드시 리턴하도록 구현한다.
- 필요한 함수 우선순위를 스스로 결정해서 구현한다
- 👉 **여기 함수들을 빨리 구현하는 게 목적이 아니라, 분석한 내용을 구현으로 점검하기 위한 수단으로 활용하자**

### **1. `locate(funcName, codes)`**

- 함수를 구성하는 어셈블리 코드를 이름과 문자열 배열로 전달한다.
- TEXT 영역에 0부터 코드를 한 줄씩 배치한다. 최종 위치는 저장하고 있다가 다른 함수를 배치할 때 겹치지 않도록 한다.
- 함수를 추가할 때마다 함수별 TEXT 주소 위치를 기록한다.
    - 예시)

```
locate("main", ["VAR A: INT", "VAR B: INT", "CALL foo()", "SET B=$RETURN"])
locate("foo", ["VAR K: INT", "RETURN 10"])

```

### **2. `setSize(type, length)`**

- type 별로 고유한 사이즈를 가지도록 등록한다.
    - 예시 : **`setSize("INT", 8)`** // int 타입을 8바이트 길이로 지정한다.
    - 메모리 시뮬레이션을 위해 스스로 필요한 타입을 지정해야 한다.
    - 이미 등록한 타입은 다시 사이즈를 바꿀 수 없다.
    - 사이즈는 1, 2, 4, 8, 16, 32 중에 하나만 가능하다.

### **3. `OOOOOOOO(address)` (internal only)**

- address 입력으로 들어오는 값을 스택에 쌓고 SP를 증가한다.
- 함수 이름도 스스로 정해보자

### **4. `OOOOOOOO()` (internal only)**

- 스택에 마지막에 쌓여있는 값을 버린다.
- 함수 이름도 스스로 정해보자

### **5. `alloc(type, count)` (internal only)**

- 이미 등록된 type에 대해 count만큼 반복해서 HEAP 영역 메모리를 할당하고, 힙 메모리 시작 위치 주소를 스택 영역에 추가한다.
    - 만약 해당 타입 크기가 8바이트 보다 작은 경우는 패딩을 붙여서 8바이트로 만든 후, count만큼 반복한다.
    - 예를 들어 boolean 타입을 1로 등록했고 alloc("boolean", 4)를 호출한다면 패딩을 붙여서 8바이트 단위로 4개 = 총 32바이트를 할당한다.

### **6. `free(stackAddress)` (internal only)**

- alloc 할 때 기록했던 스택 주소값을 입력으로 받는다. 스택 주소값에 있는 힙영역 고유 주소를 찾아서 해제하고 반환한다.
- 스택 주소 범위에 해당하면 해당 위치에 값으로 힙 영역을 찾아야 한다.
- **`stackAddress`**가 스택 주소 범위가 아니거나, **`stackAddress`** 값이 힙 주소 범위가 아닌 경우 **`exception`**을 발생하고 멈춘다.

### **7. `OOOOOOOOO()` [다음]**

- TEXT 영역에서 현재 PC 주소에 있는 코드를 실행한다.
- 코드를 실행할 때는 아래 **`어셈블리 언어 동작`** 규칙에 맞도록 internal 함수로 처리한다.
- 함수 이름도 스스로 정해보자

### **8. `usage()`**

- 스택 영역 전체크기, 사용중인 용량, 남은 용량과 힙 영역 전체크기, 사용중인 용량, 남은 용량을 순서대로 배열로 리턴한다.

### **9. `callstack()`**

- 현재 스택에 쌓여있는 호출 스택을 따로 관리해야 한다.
- 호출 스택에 쌓여있는 함수 이름과 주소값을 문자열로 리턴한다.
    - 예를 들어 call("foo", 0), call("bar", 1), call("dap", 2) 순서로 호출한 경우는 **`foo() 0x1AF00 -> bar() 0x1B100 -> dap() 0x1BF00`** 형태로 함수 이름과 텍스트 영역의 함수 시작 주소를 리턴한다.

### **10. `heapdump()`**

- 힙영역에서 사용중인 상태를 문자열 배열로 표현해서 리턴한다.
    - 힙 영역 정보는 타입과 주소, 할당한 길이, 해당 주소를 참조하는 스택 포인터 변수 정보도 포함한다.

### **11. `garbageCollect()`**

- 힙영역에 할당된 타입들 중에서 스택에 포인터 변수가 없는 경우를 찾아서 해제하는 동작을 한다.

### **12. `reset()`**

- 모든 스택과 힙, 텍스트 영역 공간을 비우고 초기상태로 만든다.

### **𝔹. 어셈블리 언어 동작**

### **1. `VAR $VarName : $Type`**

- 예시) **`VAR A : INT`**
- alloc() 내부 함수를 호출해서 메모리를 할당한다.
- 크기는 1로 지정한다.

### **2. `VAR $VarName : $Type[$Count]`**

- 예시) **`VAR ARR : INT[10]`**
- alloc() 내부 함수를 호출해서 count 크기만큼 메모리를 할당한다.

### **3. `CALL $FuncName`**

- 마지막 PC 실행중인 주소를 스택에 내부 함수를 호출한다.
- FuncName 함수이름에 해당하는 함수 주소로 PC를 옮기고 실행한다.
- 호출에 대해 기록하고 callstack()에서 활용한다.

### **4. `RETURN $Value`**

- CALL 호출 이후에 VAR로 증가했던 스택 공간은 모두 내부 함수를 호출해서 비운다.
- CALL에서 스택에서 저장해놓은 이전 호출 위치+다음 위치로 PC를 지정하고 이동한다.
    - 이전 호출 위치도 비운다.
    - 단, alloc()으로 생성된 힙 영역의 메모리는 free()할 수 없고 스택에 있던 포인터 변수만 삭제한다.
    - call()을 호출한 경우가 없을 경우 아무런 동작을 하지 않는다.
- RETURN 뒤에 값이 있는 경우는 스택 영역에 추가하고, 다음 위치 PC로 가기전에 특별히 **`$RETURN`** 값을 보관했다가 할당한다.

### **5. `RELEASE $VarName`**

- free() 내부 함수로 VarName에 해당하는 변수를 스택에서 찾아서 힙 영역을 해제한다.
- 스택에 포인터 변수를 비우지는 않는다.

### **6. `SET $VarName = $Value`**

- 변수 VarName에 해당하는 힙 주소에 Value 값을 지정한다.

### **7. `SET $VarName[$Index] = $Value`**

- 변수 VarName의 Index 위치에 해당하는 힙 주소에 Value 값을 지정한다. (**`VarName 힙 주소값 + (Index * Size)`**로 계산한다)
- 예시) **`SET INT[0] = 100`**

### **추가 요소**

# **예상결과 및 동작예시**

- 최소한 아래 방식으로 동작하도록 구현하도록 노력하자.
- locate() 내부 어셈블러 코드는 정상적인 코드라고 가정한다.
- 출력 양식은 스스로 결정해도 된다.
- 먼저 추가한 함수부터 시작한다.

```
setSize("INT", 4)
setSize("BOOL", 1)
locate("main", ["VAR A: BOOL[4]", "VAR B: INT", "CALL foo()", "SET B=$RETURN"])
locate("foo", ["VAR K: INT", "RETURN 10"])
다음()
usage()
다음()
다음()
heapdump()
다음()
callstack()
다음()
gabageCollect()
reset()

```

이 코드는 다음과 같이 동작해야 한다.

- BOOL 타입으로 4개를 힙에 할당하고,
- INT 타입으로 1개를 힙에 할당하고,
- foo를 호출하고,
- return 값 10을 받아서 B에 저장한다.

이와 같은 과정을 각 함수들을 호출하면서 동작하는지 확인한다.
