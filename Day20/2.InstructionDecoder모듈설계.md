# Day20 Instruction Decoder 모듈 설계

## 1. 🎯 `InstructionDecoder` 모듈/타입 설계

-   **역할과 책임:**
    -   16비트의 숫자(기계어)로 표현된 명령어를 CPU의 제어 장치가 이해할 수 있는 구조화된 정보(객체)로 변환(해독)하는 책임을 가집니다.
    -   복잡한 비트 연산을 캡슐화하여, `CPU` 모듈이 명령어의 비트 구조에 대해 알 필요가 없도록 만듭니다.
    -   순수 함수(pure function) 형태로 구현되어, 동일한 입력에 대해 항상 동일한 출력을 보장합니다.

-   **액션/동작:**
    -   `decode(instruction)`: 16비트 명령어(`instruction`)를 받아, 명령어의 종류, 레지스터, 상수 값 등을 포함하는 객체를 반환합니다.

-   **필요한 데이터/값:**
    -   `decode`: 16비트 명령어 코드 (`number`).

-   **변경되는 데이터/값:**
    -   없음 (상태를 가지지 않는 Stateless 모듈).

-   **반환 데이터/값:**
    -   `decode`: 해독된 명령어 정보를 담은 `DecodedInstruction` 객체.

## 2. 🏛️ 서브 타입과 데이터 구조 및 비트 연산 로직

### 2.1. `DecodedInstruction` (Object)

-   `decode` 함수의 반환 값 타입입니다.
    -   `type: string` (e.g., `'LOAD'`)
    -   `dst: string` (e.g., `'R1'`)
    -   `src1: string` (e.g., `'R2'`)
    -   `src2: string | number` (e.g., `'R3'` 또는 `250`)

### 2.2. 비트 연산 로직

16비트 명령어 `0b_OPCODE_DST_SRC1_UNUSED_SRC2` 구조를 파싱하기 위해 비트 마스크와 시프트 연산을 사용합니다.

-   **Opcode (상위 4비트):**
    -   `opcode = (instruction >> 12) & 0b1111;`
    -   12비트 오른쪽으로 시프트하여 Opcode 부분을 최하위 비트로 옮긴 후, `1111` 마스크로 해당 부분만 추출합니다.

-   **레지스터 (각 3비트):**
    -   `dstReg = (instruction >> 9) & 0b111;` (9-11번 비트)
    -   `src1Reg = (instruction >> 6) & 0b111;` (6-8번 비트)
    -   `src2Reg = instruction & 0b111;` (0-2번 비트)
    -   각 레지스터 비트 위치에 맞게 시프트한 후, `111` 마스크로 3비트만 추출합니다.
    -   추출된 숫자(1~7)는 `'R' + number` 형태로 변환합니다.

-   **상수 값:**
    -   **SUB (5비트):** `value = instruction & 0b11111;`
    -   **MOV (9비트):** `value = instruction & 0b111111111;`
    -   별도의 시프트 없이, 해당 비트 길이만큼의 마스크를 적용하여 값을 추출합니다.

```javascript
// 예시: decode 메서드 내부 로직
function decode(instruction) {
    const opcode = (instruction >> 12) & 0xF;

    switch (opcode) {
        case 0b0001: // LOAD
            const dst = 'R' + ((instruction >> 9) & 0x7);
            const base = 'R' + ((instruction >> 6) & 0x7);
            const offset = 'R' + (instruction & 0x7);
            return { type: 'LOAD', dst, src1: base, src2: offset };

        // ... 다른 명령어 케이스 ...

        case 0b1011: // MOV
            const dstMov = 'R' + ((instruction >> 9) & 0x7);
            const valueMov = instruction & 0x1FF; // 9비트 마스크
            return { type: 'MOV', dst: dstMov, src1: valueMov };

        default:
            return { type: 'UNKNOWN' };
    }
}
```

## 3. 🧪 대표적인 테스트 케이스

-   **`LOAD` 명령어 디코딩 테스트:**
    -   **입력:** `0b0001001010000011` (`LOAD R1, R2, R3`)
    -   **동작 확인:** 반환된 객체가 `{ type: 'LOAD', dst: 'R1', src1: 'R2', src2: 'R3' }` 와 일치하는지 확인.

-   **`STORE` 명령어 디코딩 테스트:**
    -   **입력:** `0b0011101011000100` (`STORE R5, R3, R4`)
    -   **동작 확인:** 반환된 객체가 `{ type: 'STORE', dst: 'R5', src1: 'R3', src2: 'R4' }` 와 일치하는지 확인.

-   **`ADD` 명령어 디코딩 테스트:**
    -   **입력:** `0b0111100010000101` (`ADD R4, R2, R5`)
    -   **동작 확인:** 반환된 객체가 `{ type: 'ADD', dst: 'R4', src1: 'R2', src2: 'R5' }` 와 일치하는지 확인.

-   **`SUB` 명령어 디코딩 테스트:**
    -   **입력:** `0b1010100110101111` (`SUB R4, R6, 15`)
    -   **동작 확인:** 반환된 객체가 `{ type: 'SUB', dst: 'R4', src1: 'R6', src2: 15 }` 와 일치하는지 확인.

-   **`MOV` 명령어 디코딩 테스트:**
    -   **입력:** `0b1011100011111010` (`MOV R4, #250`)
    -   **동작 확인:** 반환된 객체가 `{ type: 'MOV', dst: 'R4', src1: 250 }` 와 일치하는지 확인.

-   **알 수 없는 Opcode 테스트:**
    -   **입력:** `0b1111000000000000` (정의되지 않은 Opcode)
    -   **동작 확인:** 반환된 객체가 `{ type: 'UNKNOWN' }` 인지 확인.
