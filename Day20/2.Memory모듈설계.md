# Day20 Memory 모듈 설계

## 1. 🎯 `Memory` 모듈/타입 설계

-   **역할과 책임:**
    -   CPU가 사용하는 모든 데이터와 명령어를 저장하는 주 기억 장치의 역할을 수행합니다.
    -   지정된 주소(address)에 데이터를 쓰거나(`write`), 주어진 주소의 데이터를 읽는(`read`) 기능을 제공하는 것이 핵심 책임입니다.
    -   데이터의 영속성을 책임지지는 않으며, 시뮬레이션이 실행되는 동안에만 데이터를 메모리에 유지합니다.

-   **액션/동작:**
    -   `constructor()`: 메모리 공간을 초기화합니다.
    -   `write(address, value)`: 특정 `address`에 16비트 `value`를 저장합니다.
    -   `read(address)`: 특정 `address`에 저장된 16비트 값을 읽어서 반환합니다.
    -   `load(program)`: 프로그램(기계어 배열)을 메모리의 0번지부터 순차적으로 적재합니다.

-   **필요한 데이터/값:**
    -   `write`: 주소(`number`), 16비트 값(`number`).
    -   `read`: 주소(`number`).
    -   `load`: 16비트 기계어 코드 배열 (`number[]`).

-   **변경되는 데이터/값:**
    -   `write`, `load` 동작 시 내부 메모리(`storage`)의 상태가 변경됩니다.

-   **반환 데이터/값:**
    -   `read`: 해당 주소에 저장된 16비트 값 (`number`). 만약 값이 없으면 `0`을 반환합니다.

---

## 2. 🏛️ 서브 타입과 데이터 구조

-   **`storage` (Map 또는 Object):**
    -   메모리의 실제 저장 공간입니다. `Map<number, number>` 또는 `Object`를 사용하여 주소와 값을 매핑합니다.
    -   **Key (주소):**
        -   값 범위: 0 이상의 정수.
        -   고유한 값.
    -   **Value (데이터):**
        -   값 범위: 16비트 부호 없는 정수 (0 ~ 65535).
        -   필수 값.

---

## 3. 🧪 대표적인 테스트 케이스

-   **`write` 및 `read` 기본 동작 테스트:**
    -   **입력:** `memory.write(0x00A0, 12345)`
    -   **동작 확인:** `memory.read(0x00A0)` 호출 시 `12345`가 정확히 반환되는지 확인.

-   **존재하지 않는 주소 `read` 테스트:**
    -   **입력:** `memory.read(0xFFFF)` (아무것도 쓰지 않은 주소)
    -   **동작 확인:** `0`이 반환되는지 확인 (기본값 정책 검증).

-   **`load` 프로그램 적재 테스트:**
    -   **입력:** `memory.load([0xAAAA, 0xBBBB, 0xCCCC])`
    -   **동작 확인:**
        -   `memory.read(0)` 호출 시 `0xAAAA` 반환 확인.
        -   `memory.read(1)` 호출 시 `0xBBBB` 반환 확인.
        -   `memory.read(2)` 호출 시 `0xCCCC` 반환 확인.

-   **데이터 덮어쓰기 테스트:**
    -   **입력:**
        1.  `memory.write(0x00B0, 1111)`
        2.  `memory.write(0x00B0, 2222)`
    -   **동작 확인:** `memory.read(0x00B0)` 호출 시 최종적으로 쓰여진 값인 `2222`가 반환되는지 확인.
