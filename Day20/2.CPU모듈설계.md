# Day20 CPU 모듈 설계

## 1. 🎯 `CPU` 모듈/타입 설계

-   **역할과 책임:**
    -   CPU의 핵심 구성요소(`Registers`, `ALU`, `Memory`, `InstructionDecoder`)의 인스턴스를 생성하고 전체적인 데이터 흐름을 관장하는 **오케스트레이터(Orchestrator)** 역할을 수행합니다.
    -   `fetch` -> `decode` -> `execute`로 이어지는 명령어 처리 사이클을 관리하고 실행합니다.
    -   외부로부터 프로그램을 받아 `Memory`에 로드하고, 실행 시작을 책임집니다.

-   **액션/동작:**
    -   `constructor()`: CPU의 모든 구성요소를 초기화합니다.
    -   `loadProgram(program)`: 기계어 코드 배열(`program`)을 받아 `Memory`의 특정 주소부터 차례대로 적재합니다.
    -   `run()`: 프로그램 실행을 시작합니다. `PC`가 가리키는 명령어가 없을 때까지 `fetch` -> `execute` 사이클을 반복합니다.
    -   `fetch()`: `PC`가 가리키는 주소의 명령어를 `Memory`에서 가져오고, `PC`를 1 증가시킵니다.
    -   `execute(instruction)`: 가져온 명령어를 `InstructionDecoder`로 해석하고, 결과에 따라 ALU 연산, 메모리 접근 등 적절한 처리를 수행합니다.

-   **필요한 데이터/값:**
    -   `constructor`: 없음.
    -   `loadProgram`: 16비트 기계어 코드로 이루어진 배열 (`number[]`).
    -   `run`: 없음.
    -   `fetch`: 없음.
    -   `execute`: 16비트 명령어 코드 (`number`).

-   **변경되는 데이터/값:**
    -   `Registers`의 상태 (PC, R1-R7 값).
    -   `Memory`의 상태 (`STORE` 명령어 실행 시).

-   **반환 데이터/값:**
    -   각 메서드는 명시적인 반환 값 없이, 내부 상태를 변경하는 역할에 집중합니다.

## 2. 🏛️ 서브 타입과 데이터 구조

`CPU` 클래스 자체는 복잡한 내부 데이터 구조를 갖지 않지만, 다른 모듈과 상호작용하기 위해 다음과 같은 데이터 구조를 이해하고 사용합니다.

-   **`DecodedInstruction` (Object):** `InstructionDecoder`가 반환하는 객체입니다. `execute` 메서드는 이 객체의 내용에 따라 동작을 분기합니다.
    -   **`type` (string):** 명령어 종류 (e.g., `'LOAD'`, `'ADD'`).
        -   값 범위: `'LOAD'`, `'STORE'`, `'ADD'`, `'SUB'`, `'MOV'`, `'UNKNOWN'` 중 하나.
        -   필수 값.
    -   **`dst` (string):** 목적지 레지스터 이름 (e.g., `'R1'`).
        -   값 범위: `'R1'` ~ `'R7'`.
        -   `STORE`를 제외한 모든 유효 명령어에 존재.
    -   **`src1` (string):** 첫 번째 소스 레지스터 이름.
        -   값 범위: `'R1'` ~ `'R7'`.
        -   `LOAD`, `STORE`, `ADD`, `SUB` 명령어에 존재.
    -   **`src2` (string | number):** 두 번째 소스. 레지스터 이름 또는 상수 값.
        -   값 범위: `'R1'` ~ `'R7'` 또는 9비트/5비트 숫자.
        -   `LOAD`, `STORE`, `ADD`, `SUB`, `MOV` 명령어에 존재.

## 3. 🧪 대표적인 테스트 케이스

-   **`loadProgram` 테스트:**
    -   **입력:** `[0x1234, 0x5678]` 와 같은 프로그램 배열.
    -   **동작 확인:** `memory.read(0)` 호출 시 `0x1234`가 반환되는지, `memory.read(1)` 호출 시 `0x5678`이 반환되는지 확인.

-   **`run` 메서드 통합 테스트:**
    -   **입력:** `MOV R1, #10` -> `MOV R2, #20` -> `ADD R3, R1, R2` 로 이어지는 간단한 프로그램.
    -   **동작 확인:** `run()` 실행 후, `registers.get('R3')`를 호출했을 때 예상 결과 값인 `30`이 저장되어 있는지 확인.

-   **`fetch` & `execute` 사이클 테스트:**
    -   **입력:** `MOV R1, #5` (기계어: `0b1011001000000101`).
    -   **동작 확인:**
        1.  `cpu.fetch()`를 호출하면 `0b1011001000000101`이 반환되고, `registers.get('PC')`는 `1`이 되어야 함.
        2.  `cpu.execute(0b1011001000000101)`를 호출하면, `registers.get('R1')`의 값이 `5`로 변경되어야 함.
