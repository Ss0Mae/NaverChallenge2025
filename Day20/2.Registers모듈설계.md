# Day20 Registers 모듈 설계

## 1. 🎯 `Registers` 모듈/타입 설계

-   **역할과 책임:**
    -   CPU 내부에 존재하는 고속 저장 공간인 레지스터들의 상태를 관리하는 책임을 가집니다.
    -   프로그램 카운터(PC)와 7개의 범용 레지스터(R1-R7)의 값을 저장하고, 외부 요청에 따라 값을 읽거나 쓰는 기능을 제공합니다.

-   **액션/동작:**
    -   `constructor()`: 모든 레지스터(PC, R1-R7)를 0으로 초기화합니다.
    -   `get(registerName)`: 지정된 레지스터의 현재 값을 반환합니다.
    -   `set(registerName, value)`: 지정된 레지스터에 새로운 값을 저장합니다.
    -   `incrementPC()`: PC 레지스터의 값을 1 증가시킵니다. `fetch` 동작을 위해 특별히 제공되는 편의 메서드입니다.

-   **필요한 데이터/값:**
    -   `get`: 레지스터 이름 (`string`, e.g., `'PC'`, `'R1'`).
    -   `set`: 레지스터 이름 (`string`), 16비트 값 (`number`).

-   **변경되는 데이터/값:**
    -   `set`, `incrementPC` 동작 시 내부 레지스터 저장 공간의 상태가 변경됩니다.

-   **반환 데이터/값:**
    -   `get`: 해당 레지스터에 저장된 16비트 값 (`number`).

---

## 2. 🏛️ 서브 타입과 데이터 구조

-   **`storage` (Map 또는 Object):**
    -   레지스터들의 값을 저장하는 실제 공간입니다. `Map<string, number>` 또는 `Object`를 사용합니다.
    -   **Key (레지스터 이름):**
        -   값 범위: `'PC'`, `'R1'`, `'R2'`, `'R3'`, `'R4'`, `'R5'`, `'R6'`, `'R7'` 중 하나.
        -   고유한 문자열 값.
    -   **Value (데이터):**
        -   값 범위: 16비트 부호 없는 정수 (0 ~ 65535).
        -   필수 값.

---

## 3. 🧪 대표적인 테스트 케이스

-   **초기 상태 검증 테스트:**
    -   **입력:** `new Registers()`
    -   **동작 확인:** `registers.get('PC')` 와 `registers.get('R1')` 등 모든 레지스터의 초기 값이 `0`인지 확인.

-   **`set` 및 `get` 기본 동작 테스트:**
    -   **입력:** `registers.set('R3', 500)`
    -   **동작 확인:** `registers.get('R3')` 호출 시 `500`이 정확히 반환되는지 확인.

-   **`incrementPC` 동작 테스트:**
    -   **입력:**
        1.  `registers.set('PC', 10)`
        2.  `registers.incrementPC()`
    -   **동작 확인:** `registers.get('PC')` 호출 시 `11`이 반환되는지 확인.

-   **유효하지 않은 레지스터 접근 테스트:**
    -   **입력:** `registers.get('R8')` 또는 `registers.set('R0', 10)`
    -   **동작 확인:** 에러(Error)를 발생시키거나, `undefined`를 반환하는 등 예외 상황을 정책에 맞게 처리하는지 확인.
