## ✅ 미션 2: 비동기 이벤트 매니저 `EventManager` 구현

- [X] **기본 구조 설계**
    - [X] `EventManager` 클래스 정의
    - [X] 싱글톤 구현을 위한 `sharedInstance` 정적 메소드 및 private 인스턴스 변수 선언
    - [X] 구독자 정보를 저장할 내부 자료구조 설계 (예: `subscribers = []`)

- [X] **핵심 기능 구현**
    - [X] `add(subscriber, eventName, sender, handler)`: 구독자 정보를 내부 자료구조에 추가
    - [X] `remove(subscriber)`: `filter`를 사용하여 특정 구독자의 모든 정보를 제거한 새 배열 반환
    - [X] `description()`: `map`과 `join`을 사용하여 구독자 정보를 문자열로 변환

- [X] **이벤트 발행 기능 구현 (`postEvent`)**
    - [X] **(1단계) 동기 방식:** `postEvent` 호출 시, 구독자 목록을 `filter`로 필터링하고 `forEach`로 순회하며 핸들러를 즉시 실행
    - [X] **(2단계) 비동기 방식:** `postEventAsync` 메소드 구현. 핸들러 실행 로직을 `setTimeout(() => {...}, 0)`으로 감싸 이벤트 루프에 위임
    - [X] **(3단계) 지연 방식:** `postEventWithDelay` 메소드 구현. 핸들러 실행 로직을 `setTimeout(() => {...}, delayInMs)`으로 감싸 지연 실행
    - [X] `completed` 플래그와 `userData`를 이벤트 객체로 묶어 핸들러에 전달하는 로직 구현

- [X] **테스트 및 검증**
    - [X] `main` 프로그램을 작성하여 미션 요구사항의 5가지 구독자 시나리오 테스트
    - [X] 동기, 비동기, 지연 발행 시 로그 출력 시간과 순서를 비교하여 의도대로 동작하는지 확인
    - [X] `main` 프로그램이 비동기/지연 작업이 끝날 때까지 기다리도록 `setTimeout` 등으로 종료를 지연시키는 로직 추가
