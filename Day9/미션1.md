# **학습 목표**

- 프로세스와 스레드의 차이점을 이해하고, 스레드로 병렬 처리를 할 수 있도록 구현한다.
- 멀티 스레드 환경에서 공동 리소스에 접근하는 경우 레이스 컨디션이 발행하는 원리와 대응 방법을 구현한다.

## **왜 필요한가?**

이제 멀티 CPU + 멀티 Core 환경에서 멀티 스레드 프로그래밍을 작성하기 위해서는 운영체제가 멀티 스레드를 어떻게 스케줄링하고, 컨텍스트를 이동하고, 어느 부분에서 병목 현상이 생기는 지 알고 있어야 한다.

# **사전지식**

프로세스와 스레드 스케줄링에 대한 사전지식이 필요합니다.

# **기능요구사항**

- 의도적으로 멀티 스레드 환경에서 레이스 컨디션이 발생하도록 환경을 만들어야 한다.
- 멀티 스레드를 생성하는 방식을 학습하고 구현한다.
- 각 스레드에서 공동으로 접근하는 리소스를 읽고 쓰기를 반복하도록 구현한다.
- 대신 어떤 스레드가 언제 값을 읽고 쓰는지 검증할 수 있는 방법을 고민해서 포함해야 한다.
- 공동으로 접근하는 리소스 방식도 여러 가지 조건을 고려해야 한다.
- 명시적으로 동작 시차를 줄 수 있는 방법으로 스레드 스케줄링 타이밍을 조정해야 한다.

# **프로그래밍 요구사항**

운영체제와 언어 플랫폼에서 제공하는 멀티 스레드 API 를 적극 활용해야 한다. 다음 기능들을 확인해서 포함한다.

- 스레드 생성
- 스레드 멈춤
- 스레드 외부에서 데이터 생성 -> 스레드 내부 전달
- 스레드 내부에서 데이터 생성 -> 스레드 외부 전달
- 공동 데이터 생성 -> 스레드 읽기 또는 쓰기
- 스레드 대기 + IPC
- 스레드 종료

# **예상결과 및 동작예시**

- 동작을 확인하는 출력 화면 구성과 양식은 자신만의 스타일로 표현해도 된다.

> 단지 실행 결과를 출력하려고 노력하기 보다는, 운영체제에서 프로세스와 스레드 동작 구조를 학습하고 동작 방식을 자기만의 코드로 표현할 수 있는지 고민해야 한다.
