# **기능요구사항**

부스트캠프 밋업을 관리하는 콘솔 프로그램을 작성하려고 합니다. 주단위로 밋업 가능 일정을 준비해서 -> 초대받은 참석자들이 참석 가능 여부를 투표하고 -> 해당 밋업 마감하면 시간표와 참석자를 출력하는 프로그램을 작성하세요.

### **입력**

- 프로그램에서는 다음과 같은 함수를 구현해야 합니다. 반드시 클래스로 구현하지 않아도 무방합니다.
    - 밋업 가능 일정을 준비하는 arrange() 함수
    - 초대 받은 참석자들 투표를 수집하는 vote() 함수
    - 특정 밋업을 마감하는 close() 함수
- 프로그램에서는 밋업과 투표가 반복해서 진행된다고 가정하고 구현합니다. 입력과 출력을 꼭 표준 입출력 메뉴처럼 구현하지 않고 반복하는 것을 가정하고 구현해도 됩니다.

### **함수별 요구사항**

밋업은 arrange() 한 번 호출해서 시작하고, vote()를 여러번 호출해서 참석 가능한 시간을 업데이트하다가, close()를 한 번 호출해서 마무리합니다.

- 동시에 여러 밋업을 시작해서 진행할 수 있다고 가정하세요.

### **출력**

- close() 호출한 밋업에 대해 시간표를 출력한다.
- 출력은 콘솔에 밋업 시간표를 채워서 출력한다. (아래 동작 예시 참고)
- 채워진 상태와 비워진 상태를 구분해서 표시한다.
- 참석자 인원수를 표시한다.
- 최종적으로 유력한 시간대를 표시한다.

### **밋업 시간**

- 밋업 시간대중에서 참여 인원이 더 많은 시간부터 우선하여 출력한다.
- 만약 동일한 인원수가 참석 가능한 시간대가 있으면, 다음 시간대에 연속해서 참석 가능한 인원이 더 많은 우선권을 준다.
- 위 규칙 이외에는 더 합리적인 시간대를 선택하는 기준을 스스로 정해도 좋다.

# **프로그래밍 요구사항**

- VSCode 에서 작업해도 node 에서 실행가능한 형태로 작업한다.
- 함수를 사용하며, 별도 타입을 선언하지 않아도 된다.
- 함수 하나가 10줄 이상 넘어가면 하위 함수로 분리하고, 상위 함수는 하위 함수를 호출한다.
- 함수 선언 이후에 들여쓰기 단계가 3단계를 초과하는 경우는 별도 함수로 분리한다.
- 입력값을 받기 위해서 표준 입력 방식을 사용해도 되고, 소스코드 파일 안에서 함수를 호출하면서 입력값을 넣어도 된다.
- 매개변수 이름과 타입, 출력 형식은 스스로 필요성과 근거를 가지고 결정한다.
- 결과는 우선 VS Code 콘솔에 출력되어야 한다.
- 최종적으로 콘솔에 출력한 결과를 이미지로 캡처한다.
- 요구사항을 분석해서 스스로 할 일 체크리스트를 README.md에 작성한다.
- 체크리스트를 하나씩 완료할 때마다 소스와 실행 결과는 gist에 commit 한다. (Push는 한 번만 해도 무방하다.)
