# **학습 목표**

- 불변성 Immutable 값이나 부작용이 없는 변수를 적극 활용할 수 있다.
- 함수가 참조 투명성을 지키고, 부작용을 줄일 수 있도록 구현할 수 있다.
- 순수함수 Pure Function 으로 구현할 수 있다.

## **왜 필요한가?**

함수형 프로그래밍 패러다임은 객체지향 프로그래밍 패러다임과 함께 불변성, 참조투명성 등을 도입할 수 있도록 도와준다.

특히 객체 단위로 설계하더라도 함수 단위 사고의 틀을 확장해주고 자바스크립트, 스위프트, 코틀린 같은 현대 언어를 이해하기 위해 꼭 학습해야 한다.

# **기능요구사항**

부스트캠프 가상 라이브러리에서는 책을 대여해주고 관리하는 프로그램이 필요하다. 아래 요구사항을 만족하는 도서 관리 프로그램 **`CampLibrary`**을 만들어야 한다.

- 프로그램 전체 동작을 포괄하는 타입을 구현하면서 단계적으로 반복적으로 커밋한다.
    - 불변성을 가지는 데이터 구조를 구현한다.
    - 참조투명성을 가지는 타입과 함수를 고려한다.
    - 순수함수 형태를 위해서 함수형 표현으로 선언적으로 개선한다.

### **도서 관리 기능 요구사항**

- **`add(books)`** : 새로운 책을 도서관에 입고한다
- **`rent(book, person)`** : 특정 person이 특정 책 book을 대여요청한다. 재고에 따라 불가능할 수도 있다
- **`return(book, person)`** : 특정 person이 빌려간 책을 반납한다
- **`sortByDate()`** : 발행년도 순으로 전체 데이터 배열로 리턴한다
- **`top10rent()`** : 전체 대여 인원을 기준으로 상위 10개 제목을 배열로 리턴한다
- **`findBy(author)`** : 저자 이름을 기준으로 책 제목을 배열로 리턴한다
- **`totalRent()`** : 현재 대여중인 도서 정보와 대여 권수를 해시맵(딕셔너리)로 리턴한다

### **3) 도서 정보**

- 제목(필수)
- 발행년도(필수)
- 저자(필수)
- 출판사(필수)
- ISBN(필수)
- 권(선택)
- 주제분류번호(선택)

# **프로그래밍 요구사항**

- 1단계에서 만드는 불변성 데이터 구조를 적극 활용한다.
- 각 타입은 불변 타입으로 내부 속성을 바꾸지 않고, 속성을 바꾸는 경우는 새로운 인스턴스를 리턴한다.
- 시간 복잡도나 공간 복잡도 측면에서 장단점을 비교해서 구현한다

# **예상결과 및 동작예시**

프로그램의 입력과 출력을 포함해서 각 함수의 동작을 확인할 수 있는 방법과 동작 범위를 스스로 판단한다.

출력 양식은 리턴값에 도서 정보를 포함하도록 구분해서 보여줄 수 있으면 된다.

위 요구사항을 모두 구현하고 나서, 다음 요구사항을 추가로 구현한다.
