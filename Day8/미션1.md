# **학습 목표**

- 순수 함수 형태로 데이터 구조를 설계하고 기본 동작 알고리즘을 구현할 수 있다.
- 불변성 Immutable 값이나 부작용이 없는 변수를 적극 활용할 수 있다.
- 함수가 참조 투명성을 지키고, 부작용을 줄일 수 있도록 구현할 수 있다.

## **왜 필요한가?**

기존에 학습했던 대부분의 데이터 구조 : 연결리스트, 이진 트리 동작들은 데이터 가변성을 가정하고 만드는 방식이다. 함수형 프로그래밍 패러다임은 객체지향 프로그래밍 패러다임과 함께 불변성, 참조투명성 등을 도입할 수 있도록 도와준다. 함수형으로 동작하는 데이터 구조를 구현해 보자.

# **사전지식**

연결리스트와 해시맵 동작에 대해 선행 학습이 필요합니다.

# **기능요구사항**

이번 미션은 크게 두 단계로 나눠져 있다.

- 1단계) 함수형 데이터 구조를 구현하고,
- 2단계) 만든 데이터 구조를 활용해서 응용 프로그램을 구현한다.

### **불변성 데이터 구조 만들기**

- 불변성을 가지는 **연결리스트**와 **해시맵(딕셔너리)** 데이터 구조를 직접 만들어야 한다.
    - 연결리스트는 Array 나 List 를 사용하지 말고 직접 구현해 보자.
    - 해시맵도 자신이 만든 연결리스틑 활용해서 구현해 보자.
    - 단, 매개변수나 리턴타입에서 값을 바꾸지 않고 단지 전달하는 용도로 Array 나 List 는 사용해도 된다.
- 불변 타입으로 초기화할 때만 배열로 값을 넘길 수 있다. 추가나 삭제는 되지 않고, 새로운 값으로 복사되어야 한다.

### **연결리스트 필수 기능**

노드 구조와 매개 변수와 리턴 타입은 스스로 결정한다.

- append()
- insert(at)
- remove(at)
- item(at)
- clear()

### **해시맵(딕셔너리) 필수 기능**

- put(key, value)
- remove(key)
- get(key)
- contains(key)
- keys()

# **프로그래밍 요구사항**

- 자료 구조 역할을 하는 클래스나 프로토타입을 단계적으로 구현하면서 반복적으로 커밋한다.
    - 불변성을 가지는 데이터 구조를 구현한다.
    - 참조투명성을 가지는 타입과 함수를 구현한다.
    - 순수함수 형태를 위해서 함수형 표현으로 선언적으로 작성한다.
    - 해시맵의 경우 베이직 단계에서 작성했던 함수나 메소드를 함수형으로 개선해도 된다. 대신 함수 단위로 변환 과정을 나눠서 기록해야 한다.
- 각 타입은 불변 타입으로 내부 속성을 바꾸지 않고, 속성을 바꾸는 경우는 새로운 인스턴스를 리턴한다.
- 반복문 대신에 함수형 고차함수 map, filter, reduce를 적극적으로 활용한다.
- 구현한 함수를 실행(run)하는 대신 테스트 라이브러리 테스트 함수로 점검해 본다.
