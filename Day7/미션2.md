# **기능요구사항**

## **테스트 코드 작성**

- 구현한 함수들은 테스트코드를 작성해서 요구사항대로 동작하는 지 확인해야 합니다.
- 코드 생성형 AI 도구를 활용해서 테스트 명세를 주고 테스트 코드를 작성해도 됩니다.
- 명세가 바뀌면 테스트 코드도 따라서 리팩토링이 필요할 수 있습니다.
- 어떤 기능부터 언제 테스트 코드를 작성하고 구현하면 좋을지 테스트 계획을 세우고 개발합니다.
    - 함수를 만들 때마다 테스트 코드도 구현해서 점검하기 vs. 함수 구현은 다 해놓고 한꺼번에 테스트 코드를 구현해서 점검하기 vs. 테스트 코드를 먼저 만들어 놓고 해당하는 함수를 구현하기 등등

# 참고자료

## **Software Test**

### **테스트는 의도한대로 동작하는 지 검증하는 과정을 말한다.**

software test에는 unit test를 포함해서 여러가지 테스트 개념이 있다.

![](http://public.codesquad.kr/jk/unittest/v-pattern.png)

아래 링크의 4가지 레벨이 일반적인 테스트 종류로 본다.

http://www.seguetech.com/the-four-levels-of-software-testing/

- 단위 테스트 Unit test : 소프트웨어의 최소단위, 보통 함수를 가리킴
- 통합 테스트 Integration test : 단위 기능이 합쳐진 기능에 대한 테스트
- 시스템 테스트 System test : 위 내용보다 더 큰 개념, 전체 시스템에 대한 동작 테스트
- 인수 테스트 Acceptance Test : 고객이 ok할 수 있는지 판단하기 위한 테스트

![위키에 정의된 V 패턴](http://public.codesquad.kr/jk/unittest/v-pattern-new.png)

그리고 이와 별개로 아래 두가지 테스트가 있다.

- UI test : FE나 모바일 분야에서 UI 기능 단위로 진행하는 테스트. 보통 Unit test와 system test 사이라고 볼 수 있다.
- E2E test : End-to-end 테스트. 이 역시 UI 테스트와 같이 말하는 경우도 있고, 전체 시스템관점에서의 테스트로 보는 경우도 있다.

### **Unit Test**

프로그래밍의 최소단위를 테스트 하는 것으로, 그 대상은 보통 함수이다.

사람이 직접 모든 함수를 호출하는 것은 불가능하다.

특정 함수를 호출하도록 테스트 코드를 구현하고, 이를 자동화해서 그 결과를 확인하자.

### **단위 테스트의 정의**

- 빠르게 수행하고 (quickly)
- 격리된 방식으로 (isolation)
- 작은 코드 조각(단위)을 검증하고 (veridate)
- 자동화된 (automatic) 진짜 코드

```
function print(arg) {
	return 'result' + arg;
}

var expected = "result yes";
console.log(expected === print('yes')); // 성공(success)
console.log(expected === print('no'));  // 실패(fail)

```

### **JavaScript unit test 라이브러리**

앞 예제와 같이 비교를 하는 것이 귀찮기 때문에,

chai, shouldjs와 같은 **assertion 라이브러리**를 통해 테스트코드를 조금 더 쉽게 구현할 수 있다.

```
//assertion에는 여러가지 함수가 있는데,
//아래 equal 메서드는 '기대값' 과 '실제값' 이 같은지를 테스트 해준다.
assert.equal(true, false);

```

Ex) http://chaijs.com/api/assert/

### **테스트 전용 도구**

Jest 나 Mocha를 활용하는 방법을 권장한다.

> jest npm 설치 예시
> 

```
npm install --save-dev jest

```

> sum.js
> 

```
function sum(a, b) {
  return a + b;
}
module.exports = sum;

```

> sum.test.js
> 

```
const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

```

> package.json
> 

```
{
  "scripts": {
    "test": "jest"
  }
}

```

> 테스트 실행
> 

```
$ npm test
PASS  ./sum.test.js
✓ adds 1 + 2 to equal 3 (5ms)

```
