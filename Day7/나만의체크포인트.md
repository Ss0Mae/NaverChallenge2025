# Day 7: 나만의 체크포인트 ✅


### 🎯 최종 목표
- **가상 파일 시스템**: CLI를 통해 파일과 디렉토리를 관리하고, 그 상태를 영속적으로 저장하는 가상 파일 시스템을 구현합니다.
- **단위 테스트**: 구현된 모든 기능이 안정적으로 동작하는지 `Jest`를 통해 검증합니다.

---

### 📝 체크포인트 목록

- [X] **1. CLI 기본 구조 구현**
    - [X] `readline` 모듈을 사용하여 사용자 입력을 받고 출력하는 기본 CLI 환경 설정
    - [X] `exit` 명령어 처리 로직 구현

- [X] **2. 인메모리 자료구조 및 초기화 기능 구현**
    - [X] `File`, `Directory` 클래스 정의 (이름, 크기, 내용 등 속성 포함)
    - [X] `FileSystem` 클래스 정의 (루트 디렉토리, 최대 용량 관리)
    - [X] `init <size>` 명령어 구현: 파일 시스템 초기화 및 루트 생성

- [X] **3. 핵심 파일/디렉토리 조작 기능 구현 (In-Memory)**
    - [X] `mkdir <path>`: 디렉토리 생성 기능 구현
    - [X] `touch <path> [content]`: 파일 생성 및 내용 쓰기 기능 구현
    - [X] `ls [path]`: 디렉토리 목록 조회 기능 구현
    - [X] `cat <path>`: 파일 내용 읽기 기능 구현

- [X] **4. 영속성 확보**
    - [X] `FileSystem`의 상태를 하나의 JSON 파일(`vfs.json`)에 저장하는 로직 구현
    - [X] 프로그램 시작 시 `vfs.json`을 읽어 상태를 복원하는 로직 구현
    - [X] 파일/디렉토리 변경 시마다 JSON 파일 자동 업데이트

- [X] **5. `import` / `export` 기능 구현**
    - [X] `import <hostPath> <vfsPath>`: 호스트 파일을 가상 파일 시스템으로 가져오기
    - [X] `export <vfsPath> <hostPath>`: 가상 파일 시스템의 파일을 호스트로 내보내기

- [X] **6. 바이너리 파일 지원 및 리팩토링**
    - [X] `import`/`export` 시 바이너리 파일을 Base64로 인코딩/디코딩하여 처리
    - [X] 전체 코드 구조 검토 및 개선 (오류 처리, 경로 파싱 등)

- [X] **7. 단위 테스트 작성 (Jest)**
    - [X] `Jest` 개발 환경 설정 (`package.json`, `jest.config.js`)
    - [X] `FileSystem` 클래스의 각 메소드에 대한 단위 테스트 작성
    - [X] 각 CLI 명령어의 동작을 검증하는 테스트 케이스 추가 (정상/예외 케이스 포함)
