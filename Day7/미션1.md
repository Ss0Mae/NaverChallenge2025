# **학습 목표**

- 운영체제에서 파일시스템이 어떻게 동작하는지 학습한다.
- 파일시스템, 디렉토리, 텍스트파일, 바이너리 파일 등의 개념에 대해 이해한다.
- 프로그래밍 언어의 라이브러리나 시스템콜로 제공되는 API를 이용해서 파일 IO 프로그래밍을 할 수 있다.

## **왜 필요한가?**

리눅스 운영체제에서 파일 시스템이 어떻게 동작하는지 확인하고, 시스템콜로 동작하는 API 호출 방식을 만들면서 이해도를 높일 수 있다.

# **사전지식**

- 일반적인 운영체제 파일시스템 역할과 I/O 프로그래밍에 대해 학습해야 합니다.
- FAT12와 같은 단순한 파일 시스템의 구조를 먼저 학습해야 합니다.

# **기능요구사항**

간단한 가상 파일 시스템을 구현하려고 합니다.

![linux-filesystem](https://lucas-image.codesquad.kr/1751691032162_linux-filesystem.png)

위 그림은 리눅스 파일 시스템의 디렉토리 (폴더) 구조 예시입니다.

파일 시스템이 지원하는 기능은 다음과 같습니다.

### **명령 입력용 CLI 제공**

제공하는 모든 기능을 처리할 수 있는 CLI를 제공합니다.

### **초기화**

입력: 파일 시스템의 최대 크기

- 파일 시스템을 초기화합니다.
- 이 파일 시스템에는 여러개의 폴더와 파일을 담을 수 있으며 각 파일 크기 전체의 합은 지정한 최대 크기를 넘을 수 없습니다.
- 초기화 시에는 루트 디렉토리 **`/`** 만 생성합니다.

### **디렉토리 생성**

입력: 현재 디렉토리, 새로운 디렉토리 이름

- 지정된 현재 디렉토리의 하위에 새로운 디렉토리를 생성합니다.
- 디렉토리에는 디렉토리 또는 파일을 담을 수 있습니다.
    - (정확하게는 디렉토리도 파일의 한 종류로 간주합니다.)

### **텍스트 파일 생성**

입력: 디렉토리명, 파일명, 파일 내용

- 입력된 내용을 기반으로 텍스트 파일을 생성합니다.
- 텍스트 파일의 확장자는 .txt로 끝나야 합니다.

### **디렉토리 읽기 기능**

입력: 디렉토리명

- 해당 디렉토리를 읽어 디렉토리 안의 파일의 정보를 출력합니다.
- 파일의 정보에는 필수적으로 이름과 용량을 포함해야 합니다.

### **텍스트 파일 읽기 기능**

입력: 디렉토리명, 텍스트 파일명

- 해당하는 텍스트 파일의 내용을 읽습니다.

### **영속성 보장 (종료 및 재시작 지원)**

프로그램 종료 후 재시작해도 이전에 저장된 디렉토리 및 파일들이 잘 저장되어 있어야 합니다.

# **프로그래밍 요구사항**

## **인메모리 방식으로 구현**

- 일단 최소한의 설계 및 기능 구현을 위해 디스크와 파일 IO를 사용하지 않고 인 메모리로 구현해 보는 것도 좋은 방법입니다.
- 이 경우 친숙한 자료구조(리스트, 맵) 등을 이용해 구현해 볼 수 있습니다.

## **언어에서 제공하는 파일 I/O 기능 이용**

- 각 프로그래밍 언어에서 제공해 주는 파일 I/O 관련 기능을 학습하고 이를 이용해서 구현합니다. 가능하면 저수준 API를 이용해서 구현하는 것을 추천합니다.

## **파일 갯수 제한**

![fat-file-system](https://lucas-image.codesquad.kr/1737429666845fat-file-system.png)

위 그림은 FAT 파일 시스템의 디스크 저장구조를 개략적으로 나타낸 것입니다. (출처: https://www.easeus.co.kr/partition-manager-software/fat-file-system-format.html)

- 실제 파일시스템을 사용하지 않고 별도의 파일을 이용해서 가상 파일 시스템을 구현합니다.
- 실제 파일 시스템과 똑같은 구조로 구현될 필요는 없습니다.
- 영속성을 보장하기 위해서는 디렉토리 정보, 파일 정보, 실제 데이터 세 종류를 디스크에 저장하고 이를 읽어와야 합니다.
- 가능하면 생성되는 실제 파일이나 디렉토리 갯수를 최소화하도록 구현합니다.
- 아래 예시를 참고할 것

```
myfs.dat: 전체 디렉토리와 파일들이 저장되는 파일, 크기는 초기화된 최대 크기를 넘을 수 없다.
myfs.dir: 디렉토리 정보를 담고 있는 파일
myfs.info: 기타 필요한 추가적인 정보를 담고 있는 파일

```

# **예상결과 및 동작예시**

### **실행 및 초기화**

```
저장된 파일 시스템이 없습니다.
파일 시스템의 최대 크기를 입력해 주세요.
my-vfs> 500M
500 메가 파일 시스템의 초기화를 완료했습니다.
my-vfs>

```

### **디렉토리 조회**

```
my-vfs> list /
파일이 없습니다.
사용 가능한 크기: 100M

```

### **디렉토리 생성**

```
my-vfs> makedir / hello
/ 디렉토리 아래 hello를 생성합니다.

```

### **파일 생성**

```
my-vfs> create /hello greeting.txt "hello, vfs"
/hello 디렉토리 아래 greeting.txt 파일을 생성했습니다.
여유 공간: 499,999,990 byte

```

### **파일 조회**

```
my-vfs> read /hello/greeting.txt

hello, vfs

my-vfs>

```

### **export**

```
my-vfs> export /hello/greeting.txt /users/honux

가상 파일시스템 /heelo/greeting.txt를 호스트의 /users/honux로 내보내기 성공했습니다.

my-vfs>

```

# **추가 요구 사항**

### **import, export 기능**

실제 파일시스템의 파일들을 import 하거나 가상파일시스템의 파일을 실제 파일시스템으로 export 하도록 구현합니다.

### **다양한 파일 포맷 지원**

txt 뿐만 아니라, 이미지 등 다양한 파일의 import 및 export를 지원하도록 구현해 봅니다.
